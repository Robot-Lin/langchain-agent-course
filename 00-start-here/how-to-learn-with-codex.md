# 如何在 IDE 里和 Codex 一起学

这套课程默认你主要通过“描述需求 + 看解释 + 要求改动 + 做复盘”的方式学习，而不是靠自己从零手写所有代码。

## 课程中的五种学习模式

### 1. 理解模式

适合学新概念。

你可以对 Codex 说：

- 用一个真实场景解释 LangGraph 的 interrupt
- 帮我比较 LangChain agent 和 Deep Agent 的差别
- 我想做一个客服 Agent，先别写代码，先给我架构图

### 2. 搭建模式

适合把概念变成项目。

你可以对 Codex 说：

- 基于当前目录帮我搭一个最小教学项目
- 先只做知识库接入，不做前端
- 把这个原型拆成 5 个小任务，每做一步都解释原因

### 3. 复盘模式

适合查漏补缺。

你可以对 Codex 说：

- 帮我检查这个方案还缺什么
- 这个 Agent 为什么容易失控
- 按初学者视角列出我现在还没理解透的点

### 4. 读码模式

适合把“AI 已经帮你做出来了”变成“你真的看懂了它怎么实现”。

你可以对 Codex 说：

- 带我按文件解释这个功能是怎么实现的
- 这一段代码分别在处理什么输入、状态和输出
- 不要重写代码，先帮我看懂这条调用链

### 5. 全项目理解模式

适合从“看懂重点代码”升级到“理解整个项目”。

你可以对 Codex 说：

- 先带我看这个项目有哪些目录和文件，再逐个解释职责
- 带我沿着一条请求把所有相关文件串起来
- 帮我解释这里具体用了 LangChain、LangGraph 或 Deep Agents 的什么能力

## 推荐的 Vibe Coding 循环

1. 描述你要解决的问题
2. 让 Codex 帮你拆成模块
3. 先决定系统边界，不急着上实现
4. 要求生成一个最小版本
5. 每一步都追问为什么
6. 每做完一步都要求解释新增代码
7. 让 Codex 带你读主路径
8. 再让 Codex 带你扫整个项目
9. 由你来审核代码是否符合需求
10. 最后复盘这一版还缺什么

## 你需要重点培养的能力

- 能把模糊需求说清楚
- 能把一个 Agent 拆成多个能力模块
- 能看懂不同框架层级的分工
- 能看懂 AI 写出的关键代码和调用逻辑
- 能看懂整个项目是如何被组织起来的
- 能判断什么时候该增加复杂度，什么时候该收缩系统

## 学习中最常见的误区

- 误把提示词优化当成 Agent 设计
- 误把多工具调用当成高质量工作流
- 误把“能跑”当成“可用”
- 误把“会生成代码”当成“理解系统”
- 误把“AI 已经写好了”当成“自己已经掌握了”
- 误把“看懂了一部分”当成“理解了整个项目”

## 建议你如何发指令

- 先说目标，再说限制
- 先说场景，再说你当前理解
- 如果不确定，就让 Codex 先给两个层级不同的方案
- 每次都让 Codex 明确说明取舍
- 如果已经有实现，就要求 Codex 先做代码导读，再继续改动
- 如果目标是理解整个项目，就明确要求它做一次“全项目扫描”
