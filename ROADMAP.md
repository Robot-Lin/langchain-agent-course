# Roadmap

这个仓库已经完成了 `Stage 1-8` 的课程骨架与项目化教学系统，接下来会继续从“框架完整”走向“内容扎实、演示可跑、社区可参与”。

## Current Direction

当前的主方向不是继续无限扩目录，而是把现有课程逐步做厚、做实、做成真正适合长期学习和协作扩建的教学仓库。

## Near-term Priorities

### 1. Deepen Stage Content

- 把每个 Stage 的概念文档扩写成更完整的教材
- 为关键概念补更多场景化例子、失败案例和对比解释
- 强化 “为什么这样设计” 而不只是 “怎么做”

### 2. Add Session-level Learning Units

- 为每个 Stage 拆出更细的 session 或 lesson
- 增加建议学习节奏、预估时长和阶段里程碑
- 让学习者可以按更小的单元持续推进

### 3. Expand Guided Projects

- 把现有跟学项目扩展成更细的逐步教学脚本
- 为每个项目补充架构图、数据流图和验收路径
- 让 AI 能更稳定地按知识点顺序引导搭建

### 4. Strengthen Homework Standards

- 补充课后项目的评分标准与完成示例边界
- 加强“只给需求，不给答案”的教学设计
- 强化可评审、可复盘、可迁移的验收体系

### 5. Add More Full-stack Agent Examples

- 增加从后端到前端的完整 Agent 产品案例
- 覆盖不同类型应用场景，如研究、运营、知识工作、工作流审批与多工具协作
- 增加 Agent UX、审批流、状态可视化与多角色协同案例

## Mid-term Priorities

### 1. Runnable Reference Implementations

- 为代表性项目补最小可运行实现
- 提供“课程版”和“参考实现版”的双轨结构
- 降低学习者从文档走到实践的阻力

### 2. Evaluation and Observability Packs

- 为重点项目补评测方案、指标样例和观测视角
- 引导学习者从“能跑”升级到“可追踪、可评估、可治理”

### 3. Frontend Experience Patterns

- 整理常见 Agent 前端体验模式
- 增补 streaming、structured output、history、approval、trace 可视化的界面范式
- 让课程更贴近真实 Agent 产品交付

## Long-term Vision

### 1. A Living Agent Curriculum

让这个仓库成为一个可持续扩建的 Agent 学习系统，而不是静态教程。

### 2. IDE-native Learning Experience

围绕 `IDE + Codex + 文档 + 项目 + 评审` 的方式，形成一套真正适合 AI 协作学习的课程体验。

### 3. From Learning to Shipping

最终目标不是“学会概念”，而是让学习者能在 AI Agent 的帮助下，设计、构建、评测并交付一个从后端到前端的 Agent 产品。

## How To Use This Roadmap

- 如果你是学习者，可以把它当成仓库未来演进方向
- 如果你是贡献者，可以优先围绕 Near-term Priorities 提交改进
- 如果你想参与项目建设，建议先阅读 [CONTRIBUTING.md](./CONTRIBUTING.md)
