# Session Patterns

## 模式一: 概念理解会话

适用:

- 学新概念
- 比较框架
- 解释官方文档

推荐提问:

- 结合真实场景解释这个概念
- 用系统图而不是 API 说明它
- 讲讲它和相邻概念的边界

## 模式二: 课程共建会话

适用:

- 补充知识库
- 扩写某个阶段
- 丰富项目库

推荐提问:

- 基于现有课程结构扩写 Stage 2
- 给这个项目补一个详细教学包
- 把这部分拆成初阶、中阶、高阶三个层次

## 模式三: 项目推进会话

适用:

- 做原型
- 搭前端
- 补评测

推荐提问:

- 先只做最小版本
- 每做一步都解释为什么
- 先别追求完整，先把主路径跑通

## 模式四: 导师评审会话

适用:

- 自测
- 找漏洞
- 做毕业项目复盘

推荐提问:

- 站在严格评审角度指出我的架构问题
- 这个 Agent 最可能在哪些地方失败
- 这个产品的用户体验还差什么

## 每次会话的推荐结构

1. 目标
2. 当前理解
3. 限制条件
4. 想先解决的子问题
5. 希望 Codex 输出的形式
