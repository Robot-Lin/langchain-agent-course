# Agent Foundations

## 这一部分解决什么问题

这一部分帮助你从“提示词使用者”升级为“Agent 系统理解者”。

## 必须建立的基础概念

- Agent 是系统，不是单条提示词
- 模型负责生成，工具负责行动，状态负责延续，记忆负责积累，策略负责控制
- 好的 Agent 设计需要同时考虑能力、稳定性、边界、体验和可观察性

## 新手最容易混淆的点

- 把工作流和 Agent 混为一谈
- 把知识库和记忆混为一谈
- 把工具调用和任务完成混为一谈
- 把“像人在思考”误解成“系统设计合理”

## 你在这一层应该学会的事情

- 用系统图描述一个 Agent
- 判断一个需求是否真的需要 Agent
- 拆出输入、上下文、工具、状态、输出、反馈回路
- 设计最小可行版本，而不是一开始做全能系统

## 与后续模块的关系

- LangChain 负责帮你更快做出高层 Agent
- LangGraph 负责把复杂控制流和状态执行做扎实
- Deep Agents 负责把复杂任务场景进一步产品化
