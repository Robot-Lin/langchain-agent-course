# Frontend Agent Experience

这一部分聚焦一个经常被低估的事实：好的 Agent 产品不只是后端系统，也必须是好用、可信、可控的交互系统。

## 为什么前端是 Agent 必修

- Agent 会规划、调用工具、等待结果、请求澄清和触发审批
- 这些行为如果都堆在聊天消息里，用户很快就会失去控制感
- 很多 Agent 的失败其实不是模型失败，而是交互失败

## 这里会覆盖什么

- Chat UX 与 Agent UX 的差别
- Streaming 设计
- 工具执行状态可视化
- 审批与确认界面
- 历史记录与来源展示
- 任务计划面板
- 可解释反馈
- 多面板交互

## 项目里要训练的能力

- 让用户看见 Agent 正在做什么
- 让用户知道什么时候该介入
- 让用户理解结果来自模型、知识还是工具
- 让复杂任务仍然有方向感和信任感
