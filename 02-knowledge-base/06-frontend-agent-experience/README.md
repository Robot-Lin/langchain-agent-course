# Frontend Agent Experience

这部分强调一个常被低估的事实: 好的 Agent 产品不只是后端系统，也必须是好用、可信、可控的交互系统。

## 为什么前端是 Agent 必修

- Agent 会规划、调用工具、等待结果、请求澄清、触发审批
- 这些行为如果只堆在聊天消息里，用户很快就失去掌控感
- 很多 Agent 的失败其实不是模型失败，而是交互失败

## 课程会覆盖的前端主题

- Chat 界面与 Agent 界面的差别
- 流式输出设计
- 工具执行状态可视化
- 审批与确认界面
- 历史记录与记忆展示
- 任务计划面板
- 可解释反馈
- 多模态输入输出

## 项目中要训练的能力

- 让用户看见 Agent 正在做什么
- 让用户知道什么时候应该接管
- 让用户理解当前结果来自模型、知识库还是工具
- 让用户在复杂任务中保持方向感和信任感
