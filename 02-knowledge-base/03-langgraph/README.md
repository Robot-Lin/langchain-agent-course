# LangGraph

这一部分聚焦 LangGraph 作为状态化、可恢复、可人工介入的 Agent 系统编排层。

## 为什么它是核心必修

- 复杂 Agent 很快就会碰到状态、分支、恢复、中断和审批
- 只有理解 LangGraph，才会真正理解“Agent 作为系统”如何可靠运行
- 它是从 Demo 走向产品化的重要分水岭

## 这里会覆盖什么

- State 设计
- Graph 节点与边
- 条件分支与循环
- Durable execution
- Interrupts
- Human-in-the-loop
- Multi-agent orchestration

## 适合用 LangGraph 的场景

- 长任务和多步骤工作流
- 需要人工审批
- 需要失败恢复
- 需要持久执行和状态追踪
