# 课后项目: 企业 Agent 仪表盘

## 所属阶段

Stage 6: Agent Frontend

## 需求描述

请设计一个“企业 Agent 仪表盘”前端。它需要让用户查看多个任务状态、看到每个任务的计划与执行过程、审批高风险动作、浏览结构化结果，并在需要时回溯历史记录。这个界面不能只是一个聊天框，而要能支撑真实业务中的多任务、审批和状态透明。

## 约束条件

- 这一阶段必须采用 Agent UX 思维
- 必须包含状态区、审批区、结果区和历史区
- 必须考虑 streaming 或实时状态更新
- 必须明确哪些信息对用户必须可见
- 不要求落到具体前端框架实现

## 不直接给出的内容

- 不直接给完整界面布局答案
- 不直接给状态区设计答案
- 不直接给审批流设计答案

这些都需要你先自己完成。

## 你需要自己完成的部分

- 画出界面主结构
- 设计任务旅程
- 设计状态区
- 设计审批区
- 设计结果区
- 设计历史区
- 写出至少 4 个提升信任感和可控性的交互点
- 写出为什么这个产品不能只做成聊天框

## 最终效果标准

当别人读完你的方案后，应该能明确看见以下结果:

- 这个界面能够承载多任务和状态透明
- 用户知道系统现在在做什么
- 用户知道什么时候需要审批或介入
- 结果展示不是杂乱消息流，而是有结构的
- 这个前端明显提升了 Agent 的可信度和可控性

## 验收标准

你的设计至少需要满足:

- 有一张完整界面结构图
- 有一份任务旅程说明
- 有状态区、审批区、结果区和历史区说明
- 有至少 4 个提升信任感或可控性的交互点
- 有一段 220 字以上的前端设计说明
- 能清楚说明为什么它不能只是聊天框

## 如何让 Codex 辅导你

- 我现在做的是 Stage 6 课后项目，请不要直接给我完整答案

## 建议复盘问题

- 我的界面是否真的让状态可见
- 我的审批设计是否清楚
- 我的结果区是否足够结构化
- 我的界面是不是仍然太像普通聊天产品
