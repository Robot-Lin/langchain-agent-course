# 课后项目: 多源支持 Agent

## 所属阶段

Stage 5: Agent Engineering

## 需求描述

请设计一个“多源支持 Agent”。它需要能够回答支持类问题，检索帮助文档，查询部分业务数据，并在需要时调用受控工具执行支持动作。系统需要区分知识来源、用户上下文和业务数据，能够被追踪与评测，并且对高风险动作设置权限和审核机制。

## 约束条件

- 这一阶段必须采用工程化分层思维
- 必须包含知识层、工具或 MCP 层、数据层、API 层、观测层和治理层
- 必须明确区分 memory、RAG 和数据库
- 必须至少设计 3 个观测或评测点
- 必须至少设计 3 条权限或审核规则
- 不要求前端界面

## 不直接给出的内容

- 不直接给完整架构图
- 不直接给数据分层答案
- 不直接给指标和治理规则答案

这些都需要你先自己完成。

## 你需要自己完成的部分

- 画出工程分层架构
- 区分 memory、RAG 和数据库
- 设计工具或 MCP 接入方式
- 设计 API 边界
- 设计至少 3 个观测或评测点
- 设计至少 3 条权限或审核规则
- 写出为什么这个系统已经是工程问题，而不只是 Agent 设计问题

## 最终效果标准

当别人读完你的方案后，应该能明确看见以下结果:

- 这个系统的知识、数据和工具边界是清楚的
- 它知道哪些能力从哪里来
- 它知道如何被追踪和评估
- 它不会对高风险动作放任自动执行
- 它已经具备接入真实系统的工程意识

## 验收标准

你的设计至少需要满足:

- 有一张完整工程分层图
- 有清晰的 memory / RAG / 数据库区分
- 有工具或 MCP 接入说明
- 有 API 边界说明
- 有至少 3 个观测或评测点
- 有至少 3 条权限或审核规则
- 有一段 240 字以上的工程设计说明
- 能清楚说明为什么这里已经是工程问题

## 如何让 Codex 辅导你

你可以对 Codex 说:

- 我现在做的是 Stage 5 课后项目，请不要直接给我完整答案
- 你只负责检查我的工程分层、知识与数据边界、观测指标和治理规则
- 最后按验收标准帮我评分，并指出我的系统哪里还不够工程化

## 建议复盘问题

- 我的分层是否真正清楚
- 我有没有把知识和业务数据混淆
- 我的观测指标是否足够定位问题
- 我的治理规则是否覆盖了真正高风险动作
