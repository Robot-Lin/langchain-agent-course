# 课后项目: 学习教练 Agent

## 所属阶段

Stage 1: Agent Foundations

## 需求描述

请设计一个“学习教练 Agent”，面向刚进入本课程的学习者。它需要帮助用户明确学习目标、识别当前阶段、推荐下一步学习顺序，并在用户目标模糊时主动提出澄清问题。

## 约束条件

- 这一阶段不要求你写代码
- 不要求多工具联动
- 不要求长期记忆
- 必须尽量保持最小可行设计
- 必须考虑失败点和人工介入点

## 不直接给出的内容

- 不直接给你完整系统答案
- 不直接给你标准架构图
- 不直接告诉你最优工具设计

这些都需要你先自己完成。

## 你需要自己完成的部分

- 写出用户是谁
- 写出单一主任务
- 写出输入与输出
- 判断是否需要工具
- 判断是否需要状态
- 判断是否需要记忆
- 写出至少 5 个失败点
- 写出至少 3 个澄清或人工介入点
- 写出一段自然语言架构说明

## 最终效果标准

当别人读完你的方案后，应该能明确看见以下结果:

- 这个 Agent 为谁服务非常清楚
- 它的核心目标只有一个主线，不发散
- 它知道什么时候直接推荐，什么时候先追问
- 它不会假装什么都能做
- 它的输出是清晰可理解的
- 它的失败边界和人工接管点是可见的

## 验收标准

你的设计至少需要满足:

- 有清晰用户画像
- 有唯一主任务
- 有明确输入输出
- 工具设计有理由
- 状态设计有理由
- 记忆设计有理由
- 至少 5 个失败点
- 至少 3 个人工介入点
- 一段 150 字以上的完整架构说明

## 如何让 Codex 辅导你

你可以对 Codex 说:

- 我现在做的是 Stage 1 课后项目，请不要直接给我完整答案
- 你只负责问我问题、检查我的结构、指出缺失点
- 最后按验收标准帮我评分

## 建议复盘问题

- 我是不是设计得太复杂了
- 我有没有把状态、记忆、知识库混淆
- 我的失败点是不是只写了模型回答错误
- 我有没有设计用户不知道系统在做什么的情况
