# 课后项目: 多阶段研究工作流

## 所属阶段

Stage 3: LangGraph Systems

## 需求描述

请设计一个“多阶段研究工作流 Agent”。它需要接收研究主题，先生成研究计划，再按阶段收集资料、整理观点、形成总结。在关键阶段需要允许人工审阅或修正方向，若某阶段失败，需要能够恢复并继续执行，而不是从头开始。

## 约束条件

- 这一阶段必须采用 LangGraph 思维
- 必须有明确 state
- 必须至少设计 2 个 interrupt 点
- 必须有至少 2 条恢复路径
- 不要求多代理
- 不要求 Deep Agents 级复杂规划

## 不直接给出的内容

- 不直接给完整工作流图
- 不直接给 state 字段答案
- 不直接给 interrupt 位置答案

这些都需要你先自己完成。

## 你需要自己完成的部分

- 画出主路径和关键分支
- 设计 state
- 设计至少 2 个 interrupt 点
- 设计至少 2 条恢复路径
- 写出审批或人工修正机制
- 写出至少 5 个失败点
- 写出为什么这里必须使用 LangGraph

## 最终效果标准

当别人读完你的方案后，应该能明确看见以下结果:

- 这个系统是一个真正有状态的工作流，而不是普通高层原型
- 它能解释每个阶段怎么流转
- 它知道在哪里暂停等待人类输入
- 它在失败后可以恢复，而不是只能重来
- 它的控制逻辑是显式可理解的

## 验收标准

你的设计至少需要满足:

- 有一张完整工作流图
- 有至少 6 个 state 字段
- 有至少 2 个 interrupt 点
- 有至少 2 条恢复路径
- 有至少 5 个失败点
- 有一段 220 字以上的工作流说明
- 能清楚说明为什么这里必须使用 LangGraph

## 如何让 Codex 辅导你

你可以对 Codex 说:

- 我现在做的是 Stage 3 课后项目，请不要直接给我完整答案
- 你只负责检查我的工作流图、state、interrupt 和恢复策略
- 最后按验收标准帮我评分，并指出我的图哪里还不够清楚

## 建议复盘问题

- 我的 state 是否真的服务于控制流
- 我的 interrupt 是否停在正确位置
- 我的恢复路径是不是只是“重新开始”
- 我的工作流是不是已经比 Stage 2 明显更可控
