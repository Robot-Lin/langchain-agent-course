# 课后项目: 内容策略 Agent

## 所属阶段

Stage 4: Deep Agents

## 需求描述

请设计一个“内容策略 Agent”。它需要围绕一个主题做调研、分析受众、提出选题方向、规划内容结构，并给出最终内容策略建议。这个系统需要能够拆分不同子任务角色，在关键阶段允许人工调整方向，并确保最终输出始终围绕核心目标，而不是生成一堆松散内容。

## 约束条件

- 这一阶段必须采用 Deep Agents 思维
- 必须有复杂目标拆解
- 必须有上下文分层
- 必须至少设计 3 个子代理角色
- 必须有至少 3 个治理或审核点
- 不要求实现为前端产品

## 不直接给出的内容

- 不直接给完整角色设计
- 不直接给 context 分层答案
- 不直接给治理点答案

这些都需要你先自己完成。

## 你需要自己完成的部分

- 写出总目标和阶段目标
- 设计 context 分层
- 设计至少 3 个子代理角色
- 设计至少 3 个治理或审核点
- 写出至少 5 个复杂任务失控风险
- 写出为什么这里更适合 Deep Agents

## 最终效果标准

当别人读完你的方案后，应该能明确看见以下结果:

- 这个系统面对的是复杂目标，而不是简单工作流
- 它的角色分工有清晰理由
- 它的上下文不是混成一团
- 它知道在哪里需要人工纠偏
- 它的输出会围绕核心目标持续收束

## 验收标准

你的设计至少需要满足:

- 有总目标和至少 4 个阶段目标
- 有一份上下文分层说明
- 有至少 3 个子代理角色
- 有至少 3 个治理或审核点
- 有至少 5 个失控风险
- 有一段 220 字以上的复杂任务设计说明
- 能清楚说明为什么这里更适合 Deep Agents

## 如何让 Codex 辅导你

你可以对 Codex 说:

- 我现在做的是 Stage 4 课后项目，请不要直接给我完整答案
- 你只负责检查我的目标拆解、context 分层、subagents 和治理设计
- 最后按验收标准帮我评分，并指出我的复杂任务哪里还在发散

## 建议复盘问题

- 我的上下文分层是否真的减少了复杂度
- 我的子代理是否真的必要
- 我的治理点是否能防止目标偏航
- 我的设计是不是只是把 LangGraph 项目换了个名字
