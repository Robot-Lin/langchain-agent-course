# 课后项目: Agent 产品交付方案

## 所属阶段

Stage 7: Full-stack Delivery

## 需求描述

请设计一个“Agent 产品交付方案”。你可以选择一个自己前面阶段做过的 Agent 方向，把它升级成一个可交付的全栈产品。方案需要覆盖前端、后端、Agent runtime、数据或知识层、流式链路、部署、追踪和评测，并说明为什么当前技术选型合理。

## 约束条件

- 这一阶段必须采用全链路交付思维
- 必须包含前端、Agent runtime、数据或知识层、流式链路、部署、观测和评测
- 必须有一段选型说明
- 必须有明确的交付验收标准
- 不要求你真的部署上线

## 不直接给出的内容

- 不直接给完整交付图
- 不直接给选型答案
- 不直接给验收标准答案

这些都需要你先自己完成。

## 你需要自己完成的部分

- 画出全链路系统图
- 写出前后端与 runtime 链路说明
- 写出部署思路
- 写出至少 3 个追踪点
- 写出至少 3 个验收指标
- 写出一段选型说明
- 写出为什么这个方案已经具备产品交付意识

## 最终效果标准

当别人读完你的方案后，应该能明确看见以下结果:

- 这个产品不是停留在 Demo，而是具备完整交付链路
- 它知道前后端如何协同
- 它知道运行时如何被调用和反馈
- 它知道如何被追踪和评估
- 它知道为什么当前架构是合理取舍

## 验收标准

你的设计至少需要满足:

- 有一张完整全链路图
- 有一份前后端与 runtime 链路说明
- 有部署思路说明
- 有至少 3 个追踪点
- 有至少 3 个验收指标
- 有一段 240 字以上的交付说明
- 有一段清晰的选型说明

## 如何让 Codex 辅导你

你可以对 Codex 说:

- 我现在做的是 Stage 7 课后项目，请不要直接给我完整答案
- 你只负责检查我的全链路、部署、评测和选型说明
- 最后按验收标准帮我评分，并指出我的方案哪里还停留在 Demo 层

## 建议复盘问题

- 我的系统链路是否真的闭环
- 我的交付标准是否足够清楚
- 我的选型是否有明确理由
- 我的方案是不是还缺少真正可运行的一层
