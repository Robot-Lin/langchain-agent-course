# 课后项目: 结构化面试 Agent

## 所属阶段
Stage 2: LangChain Builder

## 需求描述

独立设计并实现一个结构化面试 Agent，能围绕候选人信息提出问题、整理结构化结论并提示风险。

## 约束条件

- 不要一上来把系统做成最大版本，先保证主路径清楚。
- 优先体现本阶段最核心的知识点，不为炫技增加无关复杂度。
- 如果使用 AI 生成实现，必须保留读码、解释和审核环节。

## 不直接给出的内容

- 不直接给完整最终答案
- 不跳过你的结构判断与需求澄清
- 不跳过代码理解与审核

## 你需要自己完成的部分

- 明确项目目标、用户和边界
- 设计主路径与关键能力模块
- 在需要时实现或组织相应代码
- 对关键文件、关键逻辑和关键调用链做解释
- 按标准自评并让 Codex 做最终评审

## 最终效果标准

系统应能输出结构化面试记录、候选人亮点、风险项和待确认问题，而不是只给一段自由文本。

## 验收标准

- 有明确的目标、边界和主路径
- 有与本阶段知识点对应的设计或实现
- 有至少一份结构化交付结果
- 有一份清楚的复盘或改进说明

## 代码理解与审核标准

至少能解释面试流程主函数、输出 schema、追问条件判断，以及 AI 生成代码中的控制逻辑。

## 建议如何让 Codex 辅导你

- 先帮我拆需求和主路径，不要直接给最终实现。
- 如果我已经有代码，先带我按文件读懂它，再继续修改。
- 每次新增或修改后，都先解释关键代码逻辑。
- 最后按效果标准和代码理解标准一起评审我的项目。

## 建议复盘问题

- 我这次真的学会了哪些知识点
- 这些知识点分别落到了哪些功能和代码上
- 哪段代码我还讲不清楚
- 如果要做下一版，我最该先改哪一层
