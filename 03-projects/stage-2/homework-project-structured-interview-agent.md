# 课后项目: 结构化面试 Agent

## 所属阶段

Stage 2: LangChain Builder

## 需求描述

请设计一个“结构化面试 Agent”。它需要接收岗位描述和候选人回答，动态提出后续问题，并最终输出一份结构化面试结论，供后续系统或招聘人员继续使用。

## 约束条件

- 这一阶段可以保持高层 LangChain 原型思路
- 不要求复杂多分支工作流
- 不要求长任务恢复
- 需要至少一个结构化输出设计
- 必须说明是否需要工具，以及为什么
- 必须考虑是否需要基础 human-in-the-loop

## 不直接给出的内容

- 不直接给完整原型方案
- 不直接给结构化输出字段答案
- 不直接告诉你是否必须接工具

这些都需要你先自己判断。

## 你需要自己完成的部分

- 写出主任务
- 写出输入与输出
- 判断是否需要工具
- 设计 structured output
- 写出基础控制逻辑
- 写出至少 4 个失败点
- 写出至少 2 个高风险或需要人工审核的场景
- 写出为什么当前阶段仍适合使用 LangChain

## 最终效果标准

当别人读完你的方案后，应该能明确看见以下结果:

- 这个 Agent 的主任务聚焦在“结构化面试与总结”
- 它的输出能够被人和系统继续使用
- 它知道什么时候继续追问，什么时候该收束总结
- 它没有被过早设计成复杂工作流系统
- 它对工具、审核和失败边界有清晰说明

## 验收标准

你的设计至少需要满足:

- 有唯一主任务
- 有明确输入输出
- 有一套结构化输出字段
- 工具判断有理由
- 基础控制逻辑清楚
- 至少 4 个失败点
- 至少 2 个审核场景
- 有一段 180 字以上的完整原型说明
- 能说出为什么此时用 LangChain 合适

## 如何让 Codex 辅导你

你可以对 Codex 说:

- 我现在做的是 Stage 2 课后项目，请不要直接给我完整答案
- 你只负责检查我的主任务、工具、structured output 和控制逻辑
- 最后按验收标准帮我评分，并指出哪些地方已经开始接近 LangGraph

## 建议复盘问题

- 我的 structured output 是否真的可复用
- 我的工具是不是只是为了显得更高级
- 我的控制逻辑是不是过于依赖 prompt
- 我的设计有没有已经开始逼近图式工作流
