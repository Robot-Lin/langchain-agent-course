# 统一学习闭环

为了确保学习者不只是“看了很多资料却不会做项目”，本课程每个 Stage 都采用统一的学习闭环。

这套闭环现在明确包含一条额外主线：

- 学会看懂 AI 帮你写出来的代码
- 学会把自然语言需求、系统设计和实际代码对应起来
- 学会在 Vibe Coding 过程中做“用户审核”，而不是只接受结果

## 七步学习闭环

1. 阅读知识
2. 理解方法
3. 跟做引导项目
4. 阅读并解释代码
5. 独立完成练习或课后项目
6. 接受阶段考核
7. 做复盘与迁移

## 每一步的具体作用

### 1. 阅读知识

目标:

- 建立该阶段的概念框架
- 理解这个阶段在整套课程中的位置

产出:

- 一份自然语言复述
- 一份概念关系图

### 2. 理解方法

目标:

- 学会做事流程，而不是只记名词
- 知道面对新需求时该怎么下手

产出:

- 一份方法步骤清单
- 一份决策标准清单

### 3. 跟做引导项目

目标:

- 在低风险环境中把知识转成行动
- 理解每一步为什么这么做

产出:

- 一个最小工作原型
- 一份项目拆解说明

### 4. 阅读并解释代码

目标:

- 看懂 AI 写出的代码到底在做什么
- 把需求、架构、文件、函数和运行逻辑连接起来
- 学会指出“哪段代码负责哪个功能”

产出:

- 一份代码导读说明
- 一份文件职责清单
- 一份关键函数 / 关键流程解释

### 5. 独立完成练习

目标:

- 检查是否真正掌握
- 训练迁移能力

产出:

- 至少 1 个独立完成的练习项目
- 一份自评说明

### 6. 接受阶段考核

目标:

- 防止“感觉会了”
- 建立明确放行标准

产出:

- 一份口述或书面答辩
- 一份架构说明、代码讲解或项目演示

### 7. 做复盘与迁移

目标:

- 总结哪些能力稳定了
- 明确下一阶段要补什么
- 检查是否能把方法迁移到别的场景

产出:

- 一份失误点清单
- 一份下一阶段准备清单
- 一份方法迁移说明

## 每个 Stage 的标准文件组成

每个阶段都应尽量覆盖：

- `overview`
- `concepts`
- `methods`
- `guided project`
- `challenges`
- `review checklist`
- `codex workshop`
- `session roadmap`
- `case comparisons`

如果阶段里涉及实际原型或实现，还应明确加入：

- 代码导读点
- 文件职责说明
- 关键逻辑讲解点

## 与 Codex 的配合方式

- 阅读阶段: 让 Codex 讲解、比较、举例
- 方法阶段: 让 Codex 给出结构化步骤和取舍依据
- 项目阶段: 让 Codex 一步步搭建最小原型
- 读码阶段: 让 Codex 解释文件、函数、数据流和控制流
- 练习阶段: 让 Codex 只做导师和评审，不直接给完整答案
- 复盘阶段: 让 Codex 指出盲点并生成下一轮训练任务

## 这条闭环最重要的提醒

本课程不是“自然语言提需求，AI 自动完成，用户只看结果”。

本课程要训练的是：

- 自然语言描述作为起点
- AI 编程作为过程
- 用户审核作为判断
- 最终产出完整 Agent 结果

所以学习者必须逐步学会看懂代码、审查代码、解释代码。
