# 统一学习闭环

为了确保学习者不是“看了很多资料却不会做项目”，本课程每个 Stage 都采用统一的 6 步循环。

## 六步学习循环

1. 阅读知识
2. 理解方法
3. 跟做引导项目
4. 独立完成练习
5. 接受阶段考核
6. 做复盘与迁移

## 每一步的具体作用

### 1. 阅读知识

目标:

- 建立该阶段的概念框架
- 理解这个阶段在整个课程中的位置

输出:

- 一份自然语言复述
- 一份概念关系图

### 2. 理解方法

目标:

- 学会做事流程，而不是只知道名词
- 知道面对新需求时该怎么下手

输出:

- 一份方法步骤清单
- 一份决策标准清单

### 3. 跟做引导项目

目标:

- 在低风险情境下把知识转成行动
- 理解每一步为什么这么做

输出:

- 一个最小工作原型
- 一份项目拆解说明

### 4. 独立完成练习

目标:

- 检查是否真正掌握
- 训练迁移能力

输出:

- 至少 1 个独立完成的练习项目
- 一份自评说明

### 5. 接受阶段考核

目标:

- 防止“感觉会了”
- 建立明确放行标准

输出:

- 一份口述或书面答辩
- 一份架构说明或项目演示

### 6. 做复盘与迁移

目标:

- 总结哪些能力稳定了
- 明确下一阶段需要补什么

输出:

- 一份失败点清单
- 一份下一阶段准备清单

## 每个 Stage 的标准文件组成

- `stage-x-*.md`
  包含该阶段完整教学包
- 阶段文档内部固定包括:
  概览、知识点、方法、引导项目、独立练习、考核与放行

## 与 Codex 的配合方式

- 阅读阶段: 让 Codex 讲解、比较、举例
- 方法阶段: 让 Codex 给出结构化步骤和取舍依据
- 项目阶段: 让 Codex 一步步搭建最小原型
- 练习阶段: 让 Codex 只做导师和评审，不直接给答案
- 复盘阶段: 让 Codex 指出盲点并生成下一轮训练任务
