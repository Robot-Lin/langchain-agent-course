# 统一学习闭环

为了确保学习者不是“看了很多资料却仍然无法真正理解项目”，这套课程的每一个 Stage 都采用同一套学习闭环。

这条闭环现在有两条并行主线：

- `主路径理解`
  先看懂系统最核心的一条请求如何从输入走到输出。
- `全项目理解`
  再逐步把所有文件、所有关键模块、所有重要逻辑串起来，避免只懂局部。

## 八步学习闭环

1. 阅读知识
2. 理解方法
3. 跟做引导项目
4. 阅读并解释关键代码
5. 扫描并解释整个项目
6. 独立完成练习或课后项目
7. 接受阶段考核
8. 做复盘与迁移

## 每一步的具体作用

### 1. 阅读知识

目标：

- 建立该阶段的概念框架
- 理解这一章在整套课程里的位置

产出：

- 一份自然语言总结
- 一份概念关系说明

### 2. 理解方法

目标：

- 学会一套做事方法，而不是只记名词
- 知道面对新需求时应该如何下手

产出：

- 一份方法步骤清单
- 一份判断标准清单

### 3. 跟做引导项目

目标：

- 把知识点转成实际系统设计
- 在低风险环境中理解“为什么这样搭”

产出：

- 一个最小原型或原型说明
- 一份项目拆解说明

### 4. 阅读并解释关键代码

目标：

- 先看懂项目主路径上的关键文件、关键函数、关键判断
- 把知识点和关键实现连起来

产出：

- 一份关键文件职责表
- 一份主路径调用链解释
- 一份关键逻辑讲解

### 5. 扫描并解释整个项目

目标：

- 不只看重点代码，而是逐步理解整个项目如何组织
- 确保自己知道每个主要文件为什么存在
- 把文件内部逻辑、跨文件调用关系和框架能力串起来

产出：

- 一份全项目文件地图
- 一份全项目读码清单
- 一份跨文件请求流说明
- 一份框架能力映射表

### 6. 独立完成练习或课后项目

目标：

- 检查自己是否真的掌握
- 训练迁移能力

产出：

- 至少一个独立完成的练习或课后项目
- 一份自评说明

### 7. 接受阶段考核

目标：

- 防止“感觉会了”
- 建立明确放行标准

产出：

- 一份答辩或书面回答
- 一份主路径讲解
- 一份全项目理解说明

### 8. 做复盘与迁移

目标：

- 总结哪些能力已经稳定
- 明确下一阶段最该补什么
- 检查自己能否把方法迁移到别的场景

产出：

- 一份失误点清单
- 一份下一阶段准备清单
- 一份迁移说明

## 每个 Stage 的标准文件组成

每个阶段应尽量覆盖：

- `overview`
- `concepts`
- `methods`
- `guided project`
- `challenges`
- `review checklist`
- `codex workshop`
- `session roadmap`
- `case comparisons`

如果阶段里涉及实际原型或实现，还应尽量加入：

- 关键代码导读
- 全项目文件地图
- 主路径调用链
- 框架能力映射
- 全项目审核清单

## 与 Codex 的配合方式

- 阅读阶段：让 Codex 讲解、比较、举例
- 方法阶段：让 Codex 给结构化步骤和取舍理由
- 项目阶段：让 Codex 一步步搭最小原型
- 读码阶段：让 Codex 解释文件、函数、数据流和控制流
- 全项目扫描阶段：让 Codex 按目录逐文件讲解职责和连接关系
- 练习阶段：让 Codex 做导师和评审，不直接给完整答案
- 复盘阶段：让 Codex 指出盲点并生成下一轮训练任务

## 这条闭环最重要的提醒

这套课不是：

`自然语言提需求 -> AI 自动完成 -> 用户只看结果`

这套课要训练的是：

- 自然语言描述作为起点
- AI 编程作为过程
- 用户审核作为判断
- 最终产出完整 Agent 结果
- 并且用户能解释整个项目是如何实现的
