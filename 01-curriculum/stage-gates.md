# 阶段放行标准

每个阶段都必须满足“理解 + 产出 + 复盘”三个条件，才进入下一阶段。

## Stage 0 放行

你需要能回答:

- 我为什么要学 LangChain、LangGraph、Deep Agents，而不是只学提示词
- 我希望做哪三类 Agent
- 我准备如何在 IDE 中与 Codex 配合

## Stage 1 放行

你需要能回答:

- Agent 的最小组成是什么
- 什么情况下必须引入工具
- 什么情况下必须引入状态和记忆
- 为什么一个 Agent 项目不能只看“回答质量”

你需要完成:

- 一个最小 Agent 结构图
- 一个单 Agent 项目拆解说明

## Stage 2 放行

你需要能回答:

- LangChain 解决的核心问题是什么
- 什么问题用 LangChain 足够
- 结构化输出、工具调用、middleware 分别解决什么问题

你需要完成:

- 一个 LangChain 高层 Agent 原型
- 一次失败复盘说明

## Stage 3 放行

你需要能回答:

- 为什么要从链式思维切换到图式思维
- durable execution 的实际价值是什么
- interrupt 与 human-in-the-loop 在什么业务里是必须的

你需要完成:

- 一个包含分支和人工审批的 LangGraph 工作流
- 一份 state 设计说明

## Stage 4 放行

你需要能回答:

- Deep Agents 与普通 Agent 的结构差别是什么
- 什么时候需要 subagents
- context engineering 为什么不是“多塞点上下文”

你需要完成:

- 一个复杂任务型 Agent 原型
- 一份规划与子代理拆解说明

## Stage 5 放行

你需要能回答:

- MCP、RAG、数据库、API、观测分别位于系统哪一层
- 哪些问题应该靠知识检索解决，哪些应该靠工作流解决
- Agent 权限为什么需要治理

你需要完成:

- 一个接入知识和工具的系统图
- 一份评测与风险清单

## Stage 6 放行

你需要能回答:

- 为什么 Agent UX 不等于聊天框
- 为什么审批流、状态可视化、历史记录对 Agent 特别重要
- 哪些前端设计能显著提升信任感和可控性

你需要完成:

- 一个带状态面板或审批界面的前端原型
- 一份用户旅程说明

## Stage 7 放行

你需要能回答:

- 前端、后端、Agent runtime 和知识层之间如何协同
- 为什么流式输出、状态同步和审批界面对 Agent 产品特别关键
- 部署、观测、评测和权限治理如何影响产品可用性
- 为什么同一个 Agent 需求，不同框架会有不同最优解

你需要完成:

- 一个全栈 Agent 产品原型
- 一份部署与观测方案
- 一份框架选型对比说明

## Stage 8 放行

你需要能回答:

- 你的毕业项目为什么值得做
- 为什么它的 Agent 架构适合该场景
- 你的系统如何从前端到后端形成完整闭环
- 你如何验证它真的有价值、可靠且可持续迭代

你需要完成:

- 一个可演示的全栈 Agent 毕业项目
- 一份完整架构说明
- 一份用户旅程与交互说明
- 一份评测与复盘报告

## 毕业标准

- 能独立提出一个 Agent 产品想法
- 能把它拆成模型层、工具层、知识层、状态层、体验层和评测层
- 能说明为什么使用 LangChain、LangGraph 或 Deep Agents
- 能和 Codex 协作把它逐步构建出来
- 能交付一个从后端到前端的完整 Agent 产品原型
