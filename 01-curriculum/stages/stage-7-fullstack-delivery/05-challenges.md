# 05 Challenges

## Challenge 1: 链路判断

请为一个全栈 Agent 产品写出从用户点击到最终结果的完整链路。

要求至少包含:

- 前端
- 请求通路
- Agent runtime
- 数据或知识层
- 流式反馈
- 评测与追踪

## Challenge 2: 交付判断

请判断下面哪些项目仍然只是原型，哪些已经开始具备交付条件。

### 题目 A

只有本地脚本，没有前端，没有追踪。

### 题目 B

有前端、有流式链路、有 Agent runtime、有追踪和验收指标。

### 题目 C

有很多功能，但没有办法稳定复现和排错。

## Challenge 3: 选型练习

为一个全栈 Agent 产品写一段选型说明。

要求说明:

- 为什么主要用 LangChain / LangGraph / Deep Agents 中的某一层
- 为什么当前前端形态合理
- 为什么当前部署方式合理

## Challenge 4: 评测练习

请为一个交付中的 Agent 产品写出:

- 3 个交付验收指标
- 3 个运行追踪点
- 3 个上线后应持续关注的问题

## Challenge 5: 全栈复盘练习

拿一个你前面阶段做过的 Agent，回答:

- 如果把它真正交付出来，还缺哪几层
- 最大的交付风险在哪里
