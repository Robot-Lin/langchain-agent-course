# 06 Review Checklist

这一份文档用于 Stage 7 的阶段放行。不是确认你会不会讲架构，而是确认你是否已经具备把 Agent 系统做成交付产品的能力。

## 概念自测

如果下面任意 3 项还答不清楚，建议不要急着进入 Stage 8：

- 我能画出全栈 Agent 的完整链路。
- 我能解释 Agent Server 的交付角色。
- 我能说明流式更新如何贯穿前后端。
- 我能写出 deployment 与运行思路。
- 我能写出交付验收指标。
- 我能解释 observability 和 evaluation 如何进入产品闭环。
- 我能做一次框架选型说明。
- 我能把 Agent 从工程方案推进到产品交付。

## 结构判断自测

你应该已经能做到：

- 给一个 Agent 产品画出完整交付链路。
- 说清每一层的责任和连接关系。
- 指出交付链路中最脆弱的环节。
- 指出哪些部分仍然停留在 demo 水平。

## 运行与交付自测

你应该已经能回答：

- 系统如何部署和运行。
- 系统如何追踪和排障。
- 系统如何评测与验收。
- 系统如何进入下一版迭代。

## 表达与复述检查

请尝试不用术语堆砌，而是用自然语言解释：

- 用户如何走完整条产品链路
- 团队如何知道系统是否正常运行
- 团队如何判断这一版是否可交付
- 团队如何决定下一版该改什么

如果你能把这四件事讲清楚，就说明你已经开始具备交付视角。

## 阶段答辩题

### 必答题

1. 为什么全栈 Agent 交付不只是“前端加后端”。
2. 为什么 Agent Server 对交付层重要。
3. 为什么 deployment、observability、evaluation 属于产品交付的一部分。
4. 为什么一个系统能演示不等于它已经可以交付。

### 进阶题

1. 如果一个产品链路里没有流式反馈，会损失什么。
2. 为什么选型说明是交付的一部分。
3. 你会如何把 Stage 6 的前端和 Stage 5 的工程系统真正打通。

## 放行标准

通过 Stage 7，至少要满足：

- 完成 1 个全栈 Agent 产品交付设计。
- 完成 1 个跟学项目。
- 完成 1 个课后独立项目或综合 challenge。
- 完成 1 次围绕交付链路、部署和评测的正式复盘。

## 进入 Stage 8 前的最后确认

只有当你已经能把系统完整交付，毕业项目阶段才不会只是“再做一个更大的 Demo”。
