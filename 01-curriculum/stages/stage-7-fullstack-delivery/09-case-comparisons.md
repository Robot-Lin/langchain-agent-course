# 09 Case Comparisons

这一章最适合通过对照看清交付边界。下面这些案例，帮助你判断什么时候系统还只是原型，什么时候已经开始具备产品交付条件。

## 案例一: 本地脚本型 Agent

需求：

- 本地运行
- 单机演示
- 没有前端和完整追踪

重点判断：

- 这是实验原型，不是交付产品。
- 链路、观测和运行边界都不完整。

## 案例二: 有前端但没有运行闭环的 Agent

需求：

- 有漂亮前端
- 有基础回答
- 没有清晰 runtime、追踪和评测

重点判断：

- 体验层存在，但交付层不完整。
- 用户能用，不代表团队能维护和迭代。

## 案例三: 完整研究控制台产品版

需求：

- 有前端任务入口
- 有 Agent runtime
- 有流式反馈
- 有 observability 和 evaluation
- 有 deployment 思路

重点判断：

- 已经开始具备交付条件。
- 后续重点是稳定性、治理和迭代效率。

## 案例四: 为什么很多系统看起来强却交付不了

常见原因：

- 只有功能，没有链路。
- 只有演示，没有运行方案。
- 只有回答，没有评测。
- 只有架构图，没有验收标准。

## 本章对照练习

拿你自己的一个 Agent 想法，分别回答：

- 用户入口是什么。
- 运行入口是什么。
- 实时反馈怎么回到前端。
- 团队如何知道系统跑得稳不稳。
- 当前版本是否真的具备交付条件。

如果这五个问题你都能答清，Stage 7 就真正开始落地了。
