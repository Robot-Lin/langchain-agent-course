# 03 Methods

本文件关注 Stage 7 的全栈交付方法。

## 方法一: 全链路组装法

先按链路组装系统:

1. 用户入口
2. 前端交互
3. 请求与流式通信
4. Agent runtime
5. 数据 / 知识 / 工具层
6. 追踪 / 评测 / 运行层

这样做的原因是:

- 全栈交付的关键是链路完整，不是模块堆满

## 方法二: 可演示性优先法

先确保系统能被完整演示:

- 用户能触发
- 系统能响应
- 状态能被看见
- 结果能被理解

很多系统技术上很复杂，但演示链路不完整，这意味着它还不是真正的产品。

## 方法三: 交付三问法

每个产品版本都问:

- 能不能跑
- 能不能看见
- 能不能评估

如果缺任何一个，交付都不完整。

## 方法四: 部署前置思维

在设计阶段就考虑:

- 运行在哪里
- 如何暴露给前端
- 如何追踪和扩容
- 如何排错

## 方法五: 选型说明法

每个全栈 Agent 产品都应该写一段简短说明:

- 为什么当前主线选 LangChain / LangGraph / Deep Agents
- 为什么前端采用当前交互形式
- 为什么当前部署与追踪方式合理

## 方法六: 交付复盘法

每次复盘都问:

- 用户链路是否顺畅
- 技术链路是否闭环
- 是否能快速定位问题
- 当前架构是否还适合下一版

## Stage 7 方法目标

真正掌握本阶段，不是“知道怎么部署”，而是能把一个 Agent 系统从设计推进成真正可交付、可验证、可迭代的产品。
