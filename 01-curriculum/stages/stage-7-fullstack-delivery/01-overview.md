# 01 Overview

## 本章目标

Stage 7 的目标，是让学习者从“会设计一个系统”升级到“会把这个系统交付成一个能运行、能演示、能评估、能迭代的产品”。

完成这一章后，学习者应该能回答四个关键问题：

- 这个全栈 Agent 的完整链路是什么。
- 这个系统如何运行、部署和暴露给用户。
- 这个系统如何被追踪、评测和改进。
- 这个系统为什么是一个可交付产品，而不只是一个展示型原型。

## 为什么 Stage 7 紧接 Stage 6

Stage 6 已经解决了产品体验层。但如果没有完整交付层，系统依然可能停留在：

- 原型图
- 局部 demo
- 只能本地运行的样品
- 很难复现和迭代的工程作品

所以 Stage 7 要解决的，就是把系统真正推进到：

- 能完整跑通
- 能被用户使用
- 能被团队维护
- 能被产品继续迭代

## 这一章和官方新版文档的关系

官方新版文档把交付层拆得更明确：

- Agent Server 负责以面向 Agent 的方式暴露运行能力。
- Deployment 负责运行、管理和扩展交付环境。
- Streaming API 让前端能接收运行中反馈，而不是只拿最终答案。
- LangSmith Observability 和 Evaluation 成为产品可靠性的组成部分，而不是研究阶段的附属工具。

## 学习节奏建议

### Session 1

- 建立全链路心智模型。
- 先看清从前端输入到 Agent 运行到结果回传的完整路径。

### Session 2

- 学会全栈组装方法。
- 学会 Agent Server、streaming 和部署在交付层的作用。

### Session 3

- 做引导项目。
- 重点练全链路组装、交付验收和选型说明。

### Session 4

- 用 challenge 做迁移训练。
- 让 Codex 站在交付评审角度做反馈。

## 本章产出物

学完 Stage 7，至少应该产出：

- 一张全栈 Agent 系统图。
- 一份前后端与 Agent runtime 链路说明。
- 一份 deployment、LangSmith observability、LangSmith evaluation 方案。
- 一份产品交付验收说明。
- 一份框架选型与取舍说明。

## 进入 Stage 8 前的判断标准

如果学习者还不能清楚回答下面三件事，就不建议立刻进入毕业项目阶段：

- 这个系统如何被部署与运行。
- 这个系统如何被观测与评测。
- 这个系统如何从原型推进成可迭代产品。
