# 01 Overview

## 学习目标

Stage 7 的目标，是让学习者从“会设计一个系统”升级到“会把这个系统交付成一个能运行、能演示、能迭代的产品”。

完成这一章后，学习者需要具备以下能力:

- 能把 Stage 1-6 的内容整成完整链路
- 能解释 Agent Server、deployment、streaming、evaluation 在交付中的位置
- 能设计全栈产品从请求到结果的路径
- 能说明技术架构与产品体验如何在交付层闭环

## 为什么这一阶段紧接 Stage 6

Stage 6 已经解决了产品体验层。

但如果没有完整交付层，系统依然可能停留在:

- 原型图
- 单点演示
- 不可部署的半成品

所以 Stage 7 要解决的是:

- 如何组装成完整产品
- 如何真正运行与展示
- 如何持续评测和迭代

## 学习节奏建议

### Session 1

- 读完核心概念
- 理清全栈 Agent 的完整链路

### Session 2

- 学交付方法
- 先做一版系统组装图

### Session 3

- 跟做引导项目
- 重点练前后端、Agent Server、流式链路和评测闭环

### Session 4

- 完成课后独立项目
- 让 Codex 按标准做全栈交付评审

## 本阶段的产出物

- 一张全栈 Agent 系统图
- 一份前后端与 Agent runtime 链路说明
- 一份部署、观测、评测方案
- 一份产品交付与迭代说明

## 进入下一阶段前的判断标准

如果学习者还不能回答下面 3 个问题，就不建议进入毕业项目阶段:

- 这个系统如何被部署与运行
- 这个系统如何被观测和评测
- 这个系统如何从原型进入可迭代产品
