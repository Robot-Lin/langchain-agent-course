# 01 Overview

## 学习目标

Stage 2 的目标，是让学习者从“知道系统该怎么想”进阶到“知道高层 Agent 该怎么搭”。

完成这一章后，学习者需要具备以下能力:

- 能用 LangChain 组织一个高层 Agent
- 能解释模型、工具、输出格式和中间控制逻辑的关系
- 能在不下沉到底层图编排的情况下完成一个实用原型
- 能说明为什么这个原型现在适合 LangChain，而不是 LangGraph

## 为什么这一阶段紧接 Stage 1

Stage 1 解决的是“该不该做、怎么判断”。

Stage 2 解决的是“如果现在要做，最快、最稳的高层路径是什么”。

LangChain 适合这个位置，因为它让学习者先把注意力放在:

- 任务定义
- 工具接入
- 输出结构
- 交互控制

而不是一开始就陷入复杂的状态机设计。

## 学习节奏建议

### Session 1

- 读完核心概念
- 理解 LangChain 为什么是高层入口

### Session 2

- 学习高层 Agent 设计方法
- 做一个最小 LangChain 原型设计

### Session 3

- 跟做引导项目
- 重点练 structured output 和工具边界

### Session 4

- 完成课后独立项目
- 让 Codex 按标准做评审

## 本阶段的产出物

- 一份 LangChain 高层 Agent 设计说明
- 一份工具与输出结构清单
- 一个最小原型构建步骤
- 一份高层原型的失败复盘

## 进入下一阶段前的判断标准

如果学习者还不能回答下面 3 个问题，就不建议进入 LangGraph:

- LangChain 的高层抽象到底帮我省掉了什么
- 为什么 structured output 对 Agent 特别重要
- 什么情况下高层封装开始不够用了
