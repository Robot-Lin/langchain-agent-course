# 05 Challenges

## Challenge 1: 高层设计判断

请判断下面哪些设计适合继续用 LangChain 高层原型，哪些已经开始需要 LangGraph 思维。

### 题目 A

单工具研究助手，输出结构化摘要。

### 题目 B

系统需要多步骤审批、失败恢复和长任务暂停。

### 题目 C

结构化抽取 Agent，输入一段文本，输出固定 JSON。

### 题目 D

多阶段报告生成流程，需要反复回到前一步修订。

## Challenge 2: 输出设计练习

请选择一个场景，为它设计 structured output。

- 面试助手
- 研究助理
- FAQ 助手升级版

要求:

- 写出字段
- 写出每个字段为什么需要
- 指出哪些字段是给人看的，哪些字段是给系统继续消费的

## Challenge 3: 工具边界练习

你现在有下面 4 个工具候选:

- 搜索
- 发邮件
- 数据库读写
- 文件系统

请为一个“研究助理”只保留必要工具，并说明为什么删掉其他工具。

## Challenge 4: Middleware 设计练习

请写出一个高层 Agent 可以放进 middleware 的 3 类逻辑，并说明为什么不建议都塞进 prompt。

## Challenge 5: 原型复盘练习

假设一个 LangChain 原型出现了下面问题:

- 搜到了资料，但输出很难继续加工
- 能回答，但经常跳过澄清步骤
- 工具调用结果被直接照抄，没有总结

请分别判断它更像是哪类设计问题:

- 主任务问题
- 工具问题
- 输出问题
- 控制逻辑问题
