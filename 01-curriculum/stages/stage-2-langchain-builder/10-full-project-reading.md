# 10 Full Project Reading

这一节用于把 `Stage 2` 从“重点读码”升级到“全项目理解”。

## 目标

完成这一节后，学习者应至少能：

- 说清项目有哪些主要文件
- 解释每个文件为什么存在
- 解释一条请求如何跨文件流动
- 指出这里具体用了 LangChain 的什么能力
- 说出如果要审核整个项目，会先看哪几处连接点

## 推荐阅读顺序

1. 先看项目目录
2. 找到入口文件
3. 找到 tool、schema、middleware、状态相关文件
4. 找到输出组织或 UI 相关文件
5. 再按一条真实请求把这些文件串起来

## 建议交付物

- `project-file-map`
  列出目录、主要文件、职责、不负责的内容
- `request-walkthrough`
  讲一条请求如何从输入走到输出
- `framework-mapping`
  讲每个主要实现点对应了 LangChain 的哪个能力
- `audit-hotspots`
  列出最脆弱的 2 到 3 个连接点

## 推荐给 Codex 的说法

- 先带我从目录层面看整个项目，不要一上来只讲重点函数。
- 先找入口文件，再按请求流把相关文件串起来。
- 每讲一个文件，都告诉我它负责什么、不负责什么。
- 再告诉我这里具体用了 LangChain 的哪个能力，以及为什么这样组织。
