# 06 Review Checklist

## 自测清单

如果下面任意 3 项做不到，建议不要急着进入 Stage 3。

- 我能解释 LangChain 为什么是高层入口
- 我能解释 LangChain 和 LangGraph 的上下层关系
- 我能说明 tools 在这个项目里为什么必要
- 我能设计一个结构化输出格式
- 我能解释 middleware 在系统中的作用
- 我能说明高层人工审核为什么有价值
- 我能做一个最小 LangChain 原型设计
- 我能指出什么时候高层原型开始不够用了

## 阶段答辩题

### 必答题

1. 为什么 structured output 是 Agent 系统能力，而不只是格式美化
2. 为什么工具不应该越多越好
3. middleware 为什么比全塞进 prompt 更适合放某些控制逻辑
4. 什么情况下 LangChain 高层原型已经开始不够用

### 进阶题

1. 为什么高层原型也应该尽早考虑 human-in-the-loop
2. 为什么一个“能跑”的 LangChain 原型仍然可能不适合上线
3. 如果你要把 Stage 1 的 FAQ Agent 升级到 Stage 2，你会先补什么

## 放行标准

学习者至少要做到:

- 完成 1 个 LangChain 高层原型设计
- 完成 1 个跟学项目
- 完成 1 个课后独立项目
- 完成 1 次按标准的自我答辩或 Codex 评审

## 进入 Stage 3 前的最后确认

只有当学习者已经能意识到“高层抽象的边界在哪里”，进入 LangGraph 才不会变成盲目加复杂度。
