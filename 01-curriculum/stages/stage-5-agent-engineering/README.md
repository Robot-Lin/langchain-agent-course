# Stage 5 教学包

这是课程中把 Agent 从“可工作的系统”推进到“真实工程系统”的关键单元。它承接 Stage 4，把学习者带入知识接入、工具协议、数据层、后端接口、观测、评测和治理。

## 这一阶段学完之后应该做到什么

- 能解释 MCP、RAG、memory、数据库、API、observability、evaluation 各自解决什么问题
- 能设计一个带外部能力、知识层和治理层的 Agent 架构
- 能说明不同横向能力在系统中的边界，而不是混成一团
- 能把一个 Agent 方案推进成可接真实系统的工程设计

## 本阶段的核心问题

- MCP 和普通工具接入有什么不同
- RAG、memory、数据库三者为什么不能混用概念
- 为什么评测和观测不是“最后再补”
- 为什么真实 Agent 一定要考虑权限、审计和治理
- 后端 API 在 Agent 系统里承担什么角色

## 对整个课程的作用

Stage 5 训练的是:

- 工程化连接能力
- 知识层设计能力
- 数据和存储边界意识
- 观测与评测意识
- 权限与治理思维
