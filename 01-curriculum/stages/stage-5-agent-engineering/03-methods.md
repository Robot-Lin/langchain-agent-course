# 03 Methods

本文件关注 Stage 5 的工程化设计方法。

## 方法一: 横向能力分层法

先把系统分成几层:

- Agent 层
- 工具 / MCP 层
- 知识 / RAG 层
- Memory / 数据层
- API / 业务系统层
- Observability / Evaluation 层
- 权限 / 治理层

这样做的原因是:

- 不同能力职责不同
- 分层越清楚，后续越容易维护和扩展

## 方法二: 先画边界，再接能力

每接一种新能力，都先问:

- 它属于哪一层
- 它的输入输出是什么
- 它的风险是什么
- 它如何被追踪和审计

## 方法三: 知识接入选择法

面对知识相关需求，先判断:

- 是要记住用户偏好，还是要检索文档
- 是要查业务数据，还是要引用知识资料

只有先回答这个问题，才能决定用 memory、RAG 还是数据库。

## 方法四: 观测与评测前置法

不要等系统做完才想:

- 怎么知道它跑得好不好
- 怎么定位它哪里出错

推荐在设计阶段就写清:

- 关键指标
- 关键失败点
- 关键追踪点

## 方法五: 权限治理清单法

对于每个外部能力，都写一份最小清单:

- 谁能调用
- 调用前是否需要审核
- 是否要记录日志
- 是否有不可逆风险

## 方法六: 工程化复盘法

每次复盘都问:

- 系统是否分层清楚
- 数据是否落点清楚
- 知识边界是否清楚
- 是否能追踪关键步骤
- 是否有明确验收指标

## Stage 5 方法目标

真正掌握本阶段，不是“知道很多工程名词”，而是能把一个 Agent 设计成可接真实系统、可观测、可评测、可治理的工程方案。
