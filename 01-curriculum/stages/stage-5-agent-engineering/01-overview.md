# 01 Overview

## 学习目标

Stage 5 的目标，是让学习者从“会设计复杂 Agent”升级到“会设计可接真实系统的 Agent 工程”。

完成这一章后，学习者需要具备以下能力:

- 能为 Agent 设计知识接入与工具接入层
- 能为 Agent 设计 memory、数据库和 API 的边界
- 能说明为什么 observability 和 evaluation 必须纳入主设计
- 能设计一个带治理和权限边界的工程化架构

## 为什么这一阶段紧接 Stage 4

Stage 4 已经让学习者能够组织复杂任务。

但如果没有工程化层，复杂任务 Agent 很容易停留在“概念正确、Demo 很强、真实系统接不上”的状态。

所以 Stage 5 要补上的，是:

- 外部系统连接
- 数据与知识落点
- 评测与观测闭环
- 治理与权限边界

## 学习节奏建议

### Session 1

- 读完核心概念
- 先把 MCP、RAG、memory、DB、API 的边界理清

### Session 2

- 学工程化设计方法
- 为一个现有 Agent 方案补齐横向能力层

### Session 3

- 跟做引导项目
- 重点练知识层、工具层和评测层

### Session 4

- 完成课后独立项目
- 让 Codex 按标准做工程评审

## 本阶段的产出物

- 一张工程化 Agent 架构图
- 一份工具层与知识层说明
- 一份数据层与 API 说明
- 一份观测与评测方案
- 一份权限和治理清单

## 进入下一阶段前的判断标准

如果学习者还不能回答下面 3 个问题，就不建议进入前端 Agent 体验层:

- 系统如何知道自己接了哪些能力、知识和数据
- 系统如何被观测、评测和追踪
- 系统如何避免“功能很多但不安全也不可控”
