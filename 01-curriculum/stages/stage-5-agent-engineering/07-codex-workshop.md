# 07 Codex Workshop

这个文件不是让 Codex 替你更快做完，而是让你学会如何把 Codex 用成一位会讲解、会追问、会审查、会陪练的搭档。

## Stage 5 与 Codex 配合的核心原则

在这一章里，Codex 最适合扮演这些角色：工程系统讲解员、接入层评审员、观测与评测教练、治理审查员

一句话原则：

> 在 Stage 5，Codex 应该帮助你解释、读码、复述和审核，而不是替你跳过这些过程。

## 使用前先告诉 Codex 什么

每次开始一个会话前，建议先说明：

1. 我现在处于 Stage 5
2. 我的目标不只是做出结果，还要学会看懂代码
3. 请先解释知识点，再解释代码对应关系
4. 每次只推进一步，不要一次给完整答案
5. 如果我没讲清楚逻辑，请先追问我

一个通用开场模板：

```text
我现在在学习 LangChain Agent Course 的 Stage 5。
这一章我不只想完成结果，还想学会看懂关键代码和关键逻辑。
请你先做讲解者和评审员，不要直接给完整答案。
每次只推进一步，先解释知识点，再解释它落到哪些文件和逻辑上。
如果我没讲清楚，请先追问我再继续。
```

## 模式一：知识点讲解模式

适用场景：

- 我刚读完这一章，还没有真正理解核心能力
- 我能背术语，但讲不出自己的话
- 我知道结果，却讲不清设计理由

推荐指令：

```text
请先用这章学习者能听懂的方式解释本章核心能力，不要直接讲 API，先讲系统问题和设计取舍。
```

## 模式二：代码导读模式

适用场景：

- 已经有实现或伪代码
- 想按文件、函数、节点、组件读懂主路径
- 想知道知识点分别落到了哪些代码上

推荐指令：

```text
请沿后端主路径带我读：请求进入后，工具、检索、存储、观测分别在哪里。
```

```text
请你把这章最关键的 3 个文件或模块挑出来，分别告诉我它们负责什么，以及它们是怎么串起来的。
```

## 模式三：复述检查模式

适用场景：

- 我已经看过代码，但不确定自己是不是真的理解了
- 我想在进入下一步前先做一次口头检查

推荐指令：

```text
请你不要直接继续实现，先要求我复述这条主路径。如果我讲得不清楚，再指出我漏掉了什么。
```

## 模式四：审核评审模式

适用场景：

- 我已经有一个版本，想知道哪里最该审查
- 我想训练用户审核能力，而不是只看结果

推荐指令：

```text
请站在严格评审角度，指出这份实现最值得先检查的 3 个地方。优先关注主路径、状态、边界和失败风险，不要直接重写代码。
```

## 模式五：答辩陪练模式

适用场景：

- 我准备做阶段放行
- 我想测试自己能不能脱稿解释设计和代码

推荐指令：

```text
你来当这一章的答辩官。每次只问一个问题，优先追问我为什么这样设计、代码在哪里实现、如果审核我会先看哪里。
```

## 正确用法

- 先自己解释，再让 Codex 补充
- 先读主路径，再读局部细节
- 让 Codex 做评审和追问，而不是直接代做
- 每次结束前，让 Codex 检查你是否真的讲清了逻辑

## 错误用法

- 一上来就让 Codex 直接给完整答案
- 只让 Codex 改代码，不让它解释代码
- 看到能运行就停止，不做读码和审核
- 让 Codex 代替你做答辩和复盘

## 一次完整对练模板

```text
我正在学习 Stage 5。
这是我当前的设计或实现。
请你不要直接替我做完整答案。
先按以下顺序帮助我：
1. 解释这一步对应的知识点。
2. 告诉我这些知识点落到了哪些文件、函数、节点或组件上。
3. 带我读一遍主路径。
4. 让我自己复述，如果我讲不清楚就追问我。
5. 最后再指出这版最值得先审核的 3 个地方。
```
