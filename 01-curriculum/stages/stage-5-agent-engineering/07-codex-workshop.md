# 07 Codex Workshop

这一部分是给学习者在 Stage 5 直接拿来用的 Codex 对练脚本。

## 模式一: 工程分层讲解

可以对 Codex 说:

- 用一个真实项目解释 MCP、RAG、memory、数据库和 API 的分工
- 用企业场景解释为什么治理和审计重要
- 帮我区分 observability 和 evaluation

## 模式二: 架构补全陪练

可以对 Codex 说:

- 帮我给这个 Agent 方案补齐工程分层
- 每一步都说明对应的是哪个 Stage 5 知识点
- 优先指出我现在缺的是哪一层

## 模式三: 指标与追踪评审

可以对 Codex 说:

- 帮我检查这套指标是否真的能反映系统质量
- 哪些关键步骤应该进入 trace 和日志

## 模式四: 权限治理评审

可以对 Codex 说:

- 站在严格安全评审角度，指出这个 Agent 哪些能力需要权限和审批
- 帮我补最小治理清单

## 模式五: 课后项目导师模式

可以对 Codex 说:

- 我现在做 Stage 5 课后项目，请不要直接给我完整答案
- 你只负责检查我的分层架构、知识层、数据层、评测和治理设计
- 最后按验收标准帮我评分

## 推荐学习动作

1. 先画工程分层
2. 再区分知识、记忆、数据库
3. 再设计 API、观测和评测
4. 最后让 Codex 做治理评审
