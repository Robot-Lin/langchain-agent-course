# 06 Review Checklist

## 自测清单

如果下面任意 3 项做不到，建议不要急着进入 Stage 6。

- 我能解释 MCP 在系统里的作用
- 我能区分 memory、RAG 和数据库
- 我能画出 Agent 工程分层
- 我能说明后端 API 的边界作用
- 我能设计基本 observability 方案
- 我能设计基本 evaluation 方案
- 我能写出最小权限治理清单
- 我能把 Agent 方案推进成工程架构

## 阶段答辩题

### 必答题

1. 为什么 MCP 不只是“另一个工具”
2. 为什么 memory、知识库、数据库不能混成一个概念
3. 为什么 observability 和 evaluation 不能最后再补
4. 为什么真实 Agent 项目一定要有治理设计

### 进阶题

1. 为什么企业 Agent 特别容易踩到权限和数据边界问题
2. 如果一个系统只做了知识检索，但没有指标和追踪，它缺了什么
3. 你会如何把 Stage 4 的复杂任务 Agent 接进真实业务系统

## 放行标准

学习者至少要做到:

- 完成 1 个工程化 Agent 架构设计
- 完成 1 个跟学项目
- 完成 1 个课后独立项目
- 完成 1 次围绕分层、评测和治理的答辩或评审

## 进入 Stage 6 前的最后确认

只有当学习者已经能把 Agent 接上真实系统、知识和治理层，进入前端体验层才会真正有意义。
