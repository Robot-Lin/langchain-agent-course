# 07 Codex Workshop

这一部分是给学习者在 Stage 6 直接拿来用的 Codex 对练脚本。

## 模式一: Agent UX 讲解

可以对 Codex 说:

- 用一个真实产品解释为什么 Agent UX 不等于聊天框
- 结合 streaming、审批和状态展示讲讲好前端为什么重要
- 帮我比较单面板 chat 和多面板 Agent 控制台

## 模式二: 界面结构陪练

可以对 Codex 说:

- 帮我给这个 Agent 设计前端分区
- 每一步都说明对应的是哪个 Stage 6 知识点
- 优先指出哪些信息必须单独可见

## 模式三: 审批与状态评审

可以对 Codex 说:

- 帮我检查这个界面是否足够可控
- 哪些地方应该增加审批、状态提示或失败反馈

## 模式四: 展示结构评审

可以对 Codex 说:

- 站在用户体验角度，指出我的结果展示哪里还像“纯聊天”
- 帮我补摘要区、来源区和下一步区

## 模式五: 课后项目导师模式

可以对 Codex 说:

- 我现在做 Stage 6 课后项目，请不要直接给我完整答案
- 你只负责检查我的界面分区、状态设计、审批交互和结果展示
- 最后按验收标准帮我评分

## 推荐学习动作

1. 先画任务旅程
2. 再画界面分区
3. 再补状态和审批
4. 最后让 Codex 做体验评审
