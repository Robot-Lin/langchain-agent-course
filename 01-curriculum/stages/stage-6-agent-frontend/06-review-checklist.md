# 06 Review Checklist

这一份文档用于 Stage 6 的阶段放行。不是确认你会不会画页面，而是确认你是否已经具备把 Agent 系统翻译成产品体验的能力。

## 概念自测

如果下面任意 3 项还答不清楚，建议不要急着进入 Stage 7：

- 我能解释 Agent UX 和聊天 UX 的差别。
- 我能解释 structured output 为什么对前端很重要。
- 我能解释 streaming 在 Agent 产品里不只是视觉效果。
- 我能解释 human-in-the-loop 为什么必须有前端承接。
- 我能解释状态可视化为什么会影响信任感和控制感。
- 我能解释为什么复杂 Agent 往往需要多面板界面。

## 结构判断自测

你应该已经能做到：

- 给一个 Agent 产品画出前端分区结构。
- 说清每个区的作用和信息边界。
- 指出哪些信息必须被单独可见。
- 指出哪些内容不该继续塞在消息流里。

## 可控性交互自测

你应该已经能回答：

- 用户什么时候需要审批。
- 用户在审批前应该看见什么。
- 用户拒绝或修改后系统如何继续。
- 系统失败时用户如何理解和恢复。

## 展示与反馈自测

你应该已经能设计：

- 一套 streaming 反馈层次
- 一套结构化结果展示层次
- 一套历史与来源呈现方案
- 一套失败反馈和恢复入口

## 表达与复述检查

请尝试不用术语堆砌，而是用自然语言解释：

- 用户如何知道 Agent 当前在做什么
- 用户如何知道什么时候应该介入
- 用户如何知道结果从哪里来
- 用户如何知道系统是否失败以及怎么继续

如果你能把这四件事讲清楚，就说明你已经开始具备 Agent 产品体验视角。

## 阶段答辩题

### 必答题

1. 为什么 Agent 产品不能只有一个聊天框。
2. 为什么 streaming 在 Agent 系统里不只是视觉效果。
3. 为什么审批交互必须被前端明确表达。
4. 为什么 structured output 对前端特别重要。

### 进阶题

1. 如果用户看不见当前状态，会出现什么产品问题。
2. 为什么历史、来源和结果分区会影响信任感。
3. 你会如何把 Stage 5 的工程化 Agent 做成一个可用前端产品。

## 放行标准

通过 Stage 6，至少要满足：

- 完成 1 个前端结构设计。
- 完成 1 个跟学项目。
- 完成 1 个课后独立项目或综合 challenge。
- 完成 1 次围绕状态、审批和结果展示的正式复盘。

## 进入 Stage 7 前的最后确认

只有当你已经能把系统能力翻译成产品体验时，进入全栈交付层才真正有意义。否则所谓交付，只是在脆弱后端外面包了一层 UI。
