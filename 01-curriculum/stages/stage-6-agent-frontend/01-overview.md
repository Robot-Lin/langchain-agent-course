# 01 Overview

## 学习目标

Stage 6 的目标，是让学习者从“会设计后端 Agent 系统”升级到“会设计用户真正愿意使用的 Agent 产品体验”。

完成这一章后，学习者需要具备以下能力:

- 能为 Agent 设计不同于普通聊天框的交互结构
- 能说明为什么状态、进度、审批和历史必须可见
- 能设计流式输出和结构化结果展示
- 能解释前端如何增强可控性、信任感和方向感

## 为什么这一阶段紧接 Stage 5

Stage 5 已经让系统具备真实工程能力。

但如果这些能力都只藏在后端，用户就会:

- 不知道系统正在做什么
- 不知道什么时候该介入
- 不知道结果来自哪里
- 不知道系统失败在哪里

所以 Stage 6 要解决的是“让 Agent 变成可感知、可使用、可接管的产品”。

## 学习节奏建议

### Session 1

- 读完核心概念
- 理解 Agent UX 和普通 chat UI 的根本差异

### Session 2

- 学前端交互设计方法
- 先做一个状态化界面方案

### Session 3

- 跟做引导项目
- 重点练 streaming、状态可视化和审批交互

### Session 4

- 完成课后独立项目
- 让 Codex 按标准做产品体验评审

## 本阶段的产出物

- 一份 Agent 前端结构图
- 一份用户旅程说明
- 一份状态与审批交互说明
- 一份流式与结构化输出设计说明

## 进入下一阶段前的判断标准

如果学习者还不能回答下面 3 个问题，就不建议进入全栈交付层:

- 用户如何知道 Agent 现在在做什么
- 用户何时应该接管、审批或修正
- 前端如何把复杂后端能力变成可理解体验
