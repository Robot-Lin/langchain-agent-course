# 01 Overview

## 本章目标

Stage 6 的目标，是让学习者从“会设计后端 Agent 系统”升级到“会设计用户真正愿意使用的 Agent 产品体验”。

完成这一章后，学习者应该能回答四个关键问题：

- 用户如何知道 Agent 当前在做什么。
- 用户什么时候需要接管、审批或修正。
- 系统结果如何被稳定地展示，而不是一坨长文本。
- 复杂后端能力如何被翻译成清晰、可信、可控的前端体验。

## 为什么 Stage 6 紧接 Stage 5

Stage 5 已经让系统具备了真实工程能力，但如果这些能力全都藏在后端，用户通常会遇到这些问题：

- 不知道系统现在在干什么。
- 不知道什么时候应该等待，什么时候应该介入。
- 不知道结果来自知识、工具还是系统动作。
- 不知道失败发生在哪里，也不知道如何继续。

所以 Stage 6 要解决的，就是把“后端可运行”推进到“前端可理解、可接管、可使用”。

## 这一章和官方新版文档的关系

官方新版文档已经明确把前端交互层拆得更清楚：

- structured output 不只是后端接口能力，也是前端稳定渲染能力。
- human-in-the-loop 不只是 runtime interrupt，也必须有明确的用户界面承接。
- streaming 不只是字符逐字出现，而是实时反馈系统状态的能力。
- Agent Chat UI 是一个起点，但复杂 Agent 往往需要多面板和多状态界面。

## 学习节奏建议

### Session 1

- 建立 Agent UX 心智模型。
- 先彻底分清普通 chat UI 与 Agent 产品 UI 的差别。

### Session 2

- 学会前端分区设计。
- 学会把状态、进度、审批、来源、历史单独可见化。

### Session 3

- 做引导项目。
- 重点练 streaming、structured output 和审批界面。

### Session 4

- 用 challenge 做迁移训练。
- 让 Codex 站在产品体验评审角度做反馈。

## 本章产出物

学完 Stage 6，至少应该产出：

- 一份 Agent 前端结构图。
- 一份用户任务旅程说明。
- 一份 streaming 与状态反馈说明。
- 一份 structured output 渲染说明。
- 一份审批和接管交互说明。

## 进入 Stage 7 前的判断标准

如果学习者还不能清楚回答下面三件事，就不建议立刻进入全栈交付层：

- 用户如何看见 Agent 的状态与下一步。
- 用户如何在关键节点审批、修改或拒绝动作。
- 复杂后端能力如何被前端翻译成清晰体验，而不是继续藏在聊天泡泡里。
