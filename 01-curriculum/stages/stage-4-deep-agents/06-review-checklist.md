# 06 Review Checklist

## 自测清单

如果下面任意 3 项做不到，建议不要急着进入 Stage 5。

- 我能解释 Deep Agents 适合哪类复杂任务
- 我能解释 planning 和普通步骤列表的差别
- 我能设计 context 分层
- 我能说明 subagents 为什么必要
- 我能设计复杂任务的治理点
- 我能说明复杂任务中的人工审核位置
- 我能做一个复杂任务型 Agent 结构设计
- 我能解释为什么这个任务不只是 LangGraph 工作流

## 阶段答辩题

### 必答题

1. 为什么 context engineering 不是“塞更多上下文”
2. 为什么子代理不是越多越好
3. 为什么复杂任务系统更容易失控
4. 什么情况下 Deep Agents 比普通工作流更合适

### 进阶题

1. 如果主代理持有了所有细节，会带来什么问题
2. 为什么复杂任务里的 HITL 更重要，而不是更可省略
3. 你会如何把 Stage 3 的研究工作流升级成 Deep Agent

## 放行标准

学习者至少要做到:

- 完成 1 个复杂任务型 Agent 设计
- 完成 1 个跟学项目
- 完成 1 个课后独立项目
- 完成 1 次围绕 context、subagents、治理点的答辩或评审

## 进入 Stage 5 前的最后确认

只有当学习者已经能组织复杂任务本身，而不是只会控制流程，进入工程化整合层才会真正有效。
