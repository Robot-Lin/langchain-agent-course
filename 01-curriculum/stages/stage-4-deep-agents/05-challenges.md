# 05 Challenges

## Challenge 1: 适用性判断

请判断下面哪些需求更适合 Deep Agents。

### 题目 A

固定流程审批助手。

### 题目 B

需要长时间研究、拆分多个子任务、汇总多来源观点的研究系统。

### 题目 C

结构化抽取器。

### 题目 D

需要为大型内容选题、调研、提纲、风格整合和最终成稿提供协同支持的内容工作台。

## Challenge 2: Context 分层练习

请为一个复杂任务设计三层上下文:

- 长期资料
- 当前任务状态
- 当前子任务上下文

并说明每层为什么不能混在一起。

## Challenge 3: Subagent 设计练习

请为一个复杂项目设计至少 3 个角色，并说明:

- 为什么需要这个角色
- 它拿到什么上下文
- 它返回什么结果

## Challenge 4: 偏航治理练习

假设一个复杂任务 Agent 出现下面情况:

- 子代理开始偏离原目标
- 汇总结果越来越冗长混乱
- 最终输出看起来完整，但没有回答核心问题

请写出至少 3 个治理策略。

## Challenge 5: 框架边界练习

选一个你在 Stage 3 想做的复杂工作流，回答:

- 它是否已经更适合 Deep Agents
- 如果是，核心原因是规划、上下文还是角色分工
