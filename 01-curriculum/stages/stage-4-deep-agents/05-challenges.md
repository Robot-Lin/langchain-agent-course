# 05 Challenges

这一部分开始要求学习者不只是“理解 Deep Agents 概念”，而是主动完成复杂目标拆解、context 分层、subagent 设计、治理设计和边界判断。

建议按顺序做，不要跳题。因为 `Stage 4` 真正训练的是一个渐进过程:

- 先判断任务是否适合 Deep Agents
- 再拆目标
- 再拆上下文
- 再判断角色与治理
- 最后进入独立项目

## Challenge 1: 适用边界判断练习

### 目标

训练你判断: 哪些任务更适合 Deep Agents，哪些任务还停留在 LangGraph 或更高层原型阶段。

### 任务

请判断下面哪些场景更适合 Deep Agents。

### 题目 A

单工具研究助理，输出结构化摘要。

### 题目 B

开放研究主题，需要多阶段研究、多视角分析和中途纠偏。

### 题目 C

企业审批流，需要暂停、恢复和审批。

### 题目 D

内容策略系统，需要分析受众、拆方向、规划结构并持续收敛到最终建议。

### 你至少要回答

- 当前更适合 LangChain、LangGraph 还是 Deep Agents
- 为什么
- 最关键的判断依据是什么

## Challenge 2: 复杂目标拆解练习

### 目标

训练你把复杂目标拆成层次，而不是只写一个大目标。

### 任务

请选择一个复杂目标，为它写出:

- 总目标
- 至少 4 个阶段目标
- 最终汇总目标

### 合格标准

- 阶段目标不是动作堆砌
- 阶段之间有清楚依赖关系
- 最终汇总目标不是简单拼接

## Challenge 3: context 分层练习

### 目标

训练你把上下文工程当成结构设计，而不是“给更多上下文”。

### 任务

请选择一个复杂任务，设计一份 context 分层说明，至少覆盖:

- 长期资料
- 当前任务状态
- 当前子任务上下文
- 汇总阶段需要的上下文

### 合格标准

- 不同层次的上下文职责清楚
- 不会让所有角色共享所有信息
- 能解释为什么这样分层有助于减少复杂度

## Challenge 4: subagent 必要性练习

### 目标

训练你判断哪些子任务真的需要 subagents。

### 任务

请为一个复杂任务设计至少 3 个子任务角色，并分别说明:

- 为什么这个角色需要存在
- 它拿什么上下文
- 它交付什么结果
- 如果删掉它会怎样

### 合格标准

- 角色存在有明确理由
- 不是为了“看起来像团队”而拆角色
- 能说明角色是否真的降低了主代理复杂度

## Challenge 5: 汇总与治理练习

### 目标

训练你从“会拆任务”升级到“会收束任务”。

### 任务

请为一个复杂任务写出:

- 子任务结果如何被回收
- 最终结果如何围绕总目标收敛
- 至少 3 个治理点或审核点

### 合格标准

- 汇总机制清楚
- 治理点不是装饰
- 能看出防偏航逻辑

## Challenge 6: 复杂任务失控复盘练习

### 目标

训练你把复杂任务失控看成组织问题，而不是只看模型质量。

### 任务

假设一个复杂任务型 Agent 出现下面这些问题，请判断它更像是哪类设计问题:

- 子任务结果很多，但最后结论越来越散
- 各角色都拿到太多资料，反而重复劳动
- 中间方向偏了，但系统一直往前跑
- 子代理很多，但主代理仍然承担全部决策压力

### 你至少要回答

- 这更像目标拆解问题、context 问题、subagent 问题还是治理问题
- 为什么会发生
- 系统层该怎么改

## Challenge 7: Stage 3 升级练习

### 目标

训练你把 `Stage 3` 的图系统推进成 `Stage 4` 的复杂任务系统。

### 任务

请选择一个 Stage 3 项目，把它升级成 Deep Agents 设计，并至少补上:

- 复杂目标拆解
- context 分层
- subagent 分工
- 治理点
- 为什么这里更适合 Deep Agents

### 合格标准

- 能看出从图系统到复杂任务系统的升级
- 不是简单“在原有流程上多加几个角色名”
- 新增内容真正围绕任务组织展开

## Challenge 8: 综合小作业

### 目标

把前面练习过的能力拼起来，完成一个完整的复杂任务型 Agent 说明。

### 任务

任选一个你熟悉的开放任务，写出一份 `280-550` 字的 Deep Agents 设计说明，并附带:

- 一份复杂目标拆解图
- 一份 context 分层说明
- 至少 3 个子代理角色
- 至少 3 个治理点或审核点
- 至少 5 个失控风险点
- 一段“为什么这里更适合 Deep Agents”的边界说明

## 自评 Rubric

做完这一章后，你可以按下面 5 个维度给自己打分，每项 `1-5` 分:

- 复杂目标判断: 我能否判断什么任务适合 Deep Agents
- 拆解能力: 我能否把复杂目标拆成清楚层次
- 上下文组织: 我能否做 context 分层
- 角色分工: 我能否判断哪些 subagents 真必要
- 治理意识: 我能否设计防偏航的治理点

### 粗略参考

- `20-25` 分: 可以进入课后项目或准备进入 Stage 5
- `15-19` 分: 建议再做一轮挑战题
- `14` 分及以下: 建议回看 `02-concepts.md` 和 `03-methods.md`

## 推荐学习闭环

每完成一个 challenge，都做一次小闭环:

1. 写下你的答案。
2. 让 Codex 只做复杂任务评审。
3. 记录它指出的关键问题。
4. 用你自己的语言修正答案。
5. 总结这题帮你建立了什么复杂任务组织标准。
