# 01 Overview

## 学习目标

Stage 4 的目标，是让学习者从“会设计工作流”升级到“会组织复杂任务型 Agent”。

完成这一章后，学习者需要具备以下能力:

- 能识别什么任务适合 Deep Agents
- 能把复杂目标拆成多个角色和阶段
- 能设计上下文层次，而不是把所有信息塞进一个提示
- 能解释复杂任务为什么需要更强的规划和治理

## 为什么这一阶段紧接 Stage 3

Stage 3 教会了学习者如何显式控制流程。

但在更复杂的目标下，问题往往不只是流程，而是:

- 目标本身很复杂
- 信息量很大
- 需要不同子任务角色
- 需要持续规划和修正

这时仅仅有图还不够，还要考虑如何组织复杂任务本身。

## 学习节奏建议

### Session 1

- 读完核心概念
- 理解 Deep Agents 解决的是哪类复杂问题

### Session 2

- 学习复杂任务拆解与 context engineering
- 把一个复杂目标拆成角色和上下文层

### Session 3

- 跟做引导项目
- 重点练规划、subagents 和审核点

### Session 4

- 完成课后独立项目
- 让 Codex 按标准做复杂任务评审

## 本阶段的产出物

- 一份复杂任务拆解图
- 一份 context engineering 设计说明
- 一份子代理分工说明
- 一份复杂任务治理与风险清单

## 进入下一阶段前的判断标准

如果学习者还不能回答下面 3 个问题，就不建议进入工程化整合层:

- 为什么这个复杂任务需要上下文分层
- 为什么这里的子代理是真必要，而不是炫技
- 为什么这个 Agent 的失控点不只是“答错”
