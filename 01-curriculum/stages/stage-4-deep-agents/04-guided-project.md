# 04 Guided Project

本章的引导项目是 深度研究 Agent。这一章的重点不是把项目做得最大，而是把本章知识点真实落到功能、文件、代码、跨文件连接和审核判断上。

## 项目目标

把复杂研究任务拆成可规划、可分派、可复核的 Deep Agent 系统。

## 对应知识点

planning、context engineering、subagents、复杂任务治理、人工复核

## 知识点如何落到代码上

重点映射到这些实现层：

- planner、task queue、subagent contract、context builder、review hooks
- 主路径输入如何进入系统
- 关键状态、关键判断或关键输出在哪里形成
- 不同文件之间如何把主代理、子代理、记忆和治理层串起来

## 推荐推进顺序

1. 先定义复杂任务边界
2. 拆出主代理与子代理职责
3. 设计上下文装配规则
4. 加入人工复核点
5. 先对照 `02-concepts.md` 后半部分的代码教材和 `03-methods.md` 末尾的 API 对照附录理解最小 Deep Agent 结构
6. 完成设计后读懂 planner、subagent 和 context 的主链路
7. 再做一次整套系统和全项目扫描，解释每个主要文件、角色和治理点如何连接

## 本章必须完成的代码导读

做完项目设计或实现后，至少补做下面这轮读码：

- 先列出关键文件或关键模块
- 再说清它们各自负责什么
- 再解释一条主路径如何从输入走到输出
- 再按目录把主要文件扫一遍，说明哪些是主代理层、哪些是子代理层、哪些是 context / memory / governance 层
- 最后指出如果你来审核，会优先看哪 2 到 3 个风险点

## 最终交付物

- 任务拆解表、上下文策略说明、子代理边界表、关键逻辑讲解
- 一份主路径解释
- 一份全项目文件地图
- 一份用户审核要点清单

## 推荐给 Codex 的说法

- 先不要继续扩功能，先带我读懂 planner 和 subagent 是怎么配合的。
- 不要只告诉我结果，先告诉我每个知识点落到了哪些文件和逻辑上。
- 在主路径讲完之后，再带我按文件把整个项目扫一遍。
- 如果我讲不清楚代码逻辑，不要直接进入下一步。
