# 04 Guided Project

本章的引导项目是 深度研究 Agent。这一章的重点不是把项目做得最大，而是把本章知识点真实落到功能、文件、代码和审核判断上。

## 项目目标

把复杂研究任务拆成可规划、可分派、可复核的 Deep Agent 系统。

## 对应知识点

planning、context engineering、subagents、复杂任务治理、人工复核

## 知识点如何落到代码上

重点映射到这些实现层：

- planner、task queue、subagent contract、context builder、review hooks
- 主路径输入如何进入系统
- 关键状态、关键判断或关键输出在哪里形成

## 推荐推进顺序

1. 先定义复杂任务边界
2. 拆出主代理与子代理职责
3. 设计上下文装配规则
4. 加入人工复核点
5. 最后读懂 planner、subagent 和 context 的主链路

## 本章必须完成的代码导读

做完项目设计或实现后，至少补做下面这轮读码：

- 先列出关键文件或关键模块
- 再说清它们各自负责什么
- 再解释一条主路径如何从输入走到输出
- 最后指出如果你来审核，会优先看哪 2 到 3 个风险点

## 最终交付物

- 任务拆解表、上下文策略说明、子代理边界表、关键逻辑讲解
- 一份主路径解释
- 一份用户审核要点清单

## 推荐给 Codex 的说法

- 先不要继续扩功能，先带我读懂 planner 和 subagent 是怎么配合的。
- 不要只告诉我结果，先告诉我每个知识点落到了哪些文件和逻辑上。
- 如果我讲不清楚代码逻辑，不要直接进入下一步。
