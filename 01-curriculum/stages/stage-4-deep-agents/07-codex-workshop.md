# 07 Codex Workshop

这一部分是给学习者在 Stage 4 直接拿来用的 Codex 对练脚本。

## 模式一: 复杂任务讲解

可以对 Codex 说:

- 用一个真实项目解释 Deep Agents 为什么适合复杂任务
- 比较 LangGraph 工作流和 Deep Agent 在同一个任务里的差别
- 用具体例子解释 context engineering

## 模式二: 复杂目标拆解陪练

可以对 Codex 说:

- 帮我把这个复杂目标拆成阶段目标、子任务角色和汇总层
- 每一步都说明对应的是哪个 Stage 4 知识点

## 模式三: Context 评审

可以对 Codex 说:

- 帮我检查这个上下文分层是否合理
- 哪些信息应该给主代理，哪些应该隔离给子代理

## 模式四: 治理点评审

可以对 Codex 说:

- 站在系统治理角度，指出我的复杂任务 Agent 最容易偏航的地方
- 帮我补审批点、回收点和质量检查点

## 模式五: 课后项目导师模式

可以对 Codex 说:

- 我现在做 Stage 4 课后项目，请不要直接给我完整答案
- 先检查我的目标拆解、context 分层、subagents 和治理设计
- 最后按验收标准帮我评分

## 推荐学习动作

1. 先拆复杂目标
2. 再设计 context 分层
3. 再设计子代理角色
4. 最后让 Codex 评审是否真的适合 Deep Agents
