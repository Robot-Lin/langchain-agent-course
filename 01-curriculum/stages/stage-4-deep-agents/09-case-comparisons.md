# 09 Case Comparisons

这个文件用于训练 `Stage 4` 的关键能力: 不是一看到复杂需求就盲目上 Deep Agents，而是先判断它的难点究竟在流程控制，还是在复杂任务组织。

建议你一边读，一边回答:

- 为什么这个任务更适合或不适合 Deep Agents
- 它的难点到底落在目标、上下文、角色，还是流程
- 哪个点最需要 planning、context engineering 或 subagents

## 案例 1: 单工具研究助理

### 场景

用户提出研究问题，系统用一个工具收集资料并输出结构化摘要。

### 当前适合 Deep Agents 吗

通常不适合。

### 为什么

- 主任务仍然清楚
- 工具不多
- 难点不在复杂任务组织
- LangChain 或轻量 LangGraph 往往足够

## 案例 2: 多阶段研究工作流

### 场景

系统按阶段完成研究计划、资料收集、观点整理、总结输出，并在关键节点等待人工调整方向。

### 当前适合 Deep Agents 吗

开始适合，尤其当目标开放度高时。

### 为什么

- 不只是流程问题
- 还涉及复杂目标拆解和上下文组织
- 需要多阶段成果回收与修正

## 案例 3: 内容策略 Agent

### 场景

系统围绕品牌目标，分析受众、提出选题、规划内容结构、形成策略建议，并在关键阶段允许人工纠偏。

### 当前适合 Deep Agents 吗

很适合。

### 为什么

- 目标本身复杂且开放
- 子任务角色天然差异明显
- context engineering 是核心难点
- 最终结果要围绕总目标收敛

## 案例 4: 企业审批流

### 场景

系统接收任务、走审批、失败恢复、持续推进。

### 当前适合 Deep Agents 吗

不一定。

### 为什么

- 难点更多在显式流程控制
- 这更像 LangGraph 典型场景
- 除非任务本身还很开放，需要复杂规划与角色组织

### Stage 4 判断关键

不是因为“很复杂”就自动适合 Deep Agents，而要看复杂度主要落在哪一层。

## 案例 5: 代码库理解顾问

### 场景

系统需要理解大型代码库，拆出多个分析角度，分别调查架构、依赖、风险、热点路径，最后汇总成可用结论。

### 当前适合 Deep Agents 吗

很适合。

### 为什么

- 目标开放
- 信息量巨大
- 子任务视角差异明显
- 汇总与回收机制非常重要

## 案例 6: FAQ Agent 升级版

### 场景

FAQ Agent 现在还能推荐学习路径和做轻量澄清。

### 当前适合 Deep Agents 吗

通常不适合。

### 为什么

- 虽然比基础 FAQ 更复杂
- 但难点通常还没到复杂任务组织这一层
- 轻量 LangChain 或 LangGraph 通常已足够

## 案例 7: 深度研究 Agent

### 场景

系统围绕一个开放研究主题，拆出多个研究方向，分别收集证据、判断趋势、比较观点，最终生成综合结论并允许人类在关键节点修正方向。

### 当前适合 Deep Agents 吗

非常适合。

### 为什么

- 目标高度开放
- 任务长且多层
- 需要强 planning
- 需要 context 分层
- 需要复杂汇总与治理

## 案例 8: 多角色内容生产系统

### 场景

系统需要像小团队一样完成内容主题研究、受众判断、结构规划、风格校正和最终策略输出。

### 当前适合 Deep Agents 吗

适合，但必须克制。

### 为什么

- 角色差异明显
- 子任务之间有自然分工
- 需要统一汇总到一个总目标

### 风险点

- 很容易把角色拆太多
- 很容易把任务做成“看起来很高级”的自动公司幻想

## 一张总表看差异

| 案例 | 当前是否适合 Deep Agents | 关键原因 | 最关键的复杂任务能力 |
| --- | --- | --- | --- |
| 单工具研究助理 | 通常不适合 | 难点不在任务组织 | 无需强上 Deep Agents |
| 多阶段研究工作流 | 开始适合 | 目标与流程同时复杂 | planning + context |
| 内容策略 Agent | 是 | 目标开放且需收束 | context + governance |
| 企业审批流 | 不一定 | 难点更偏流程控制 | LangGraph 已常足够 |
| 代码库理解顾问 | 是 | 多视角分析与汇总 | subagents + synthesis |
| FAQ Agent 升级版 | 通常不适合 | 复杂度还不够高 | 高层原型或图系统即可 |
| 深度研究 Agent | 是 | 长任务、开放目标、多层结果 | planning + context + governance |
| 多角色内容生产系统 | 是，但需克制 | 角色协作与统一收束 | subagents + governance |

## 使用建议

读完这个文件后，建议你选 2 到 3 个案例，回答下面这些问题:

1. 这个任务的难点更多在流程，还是在任务组织。
2. 这个任务最需要的上下文分层是什么。
3. 哪些角色是真的必要，哪些角色只是“看起来像团队”。
4. 最终结果如何被收束到总目标，而不是越来越散。

如果你能稳定回答这些问题，`Stage 4` 的复杂任务判断就开始成形了。
