# 02 Concepts

## 1. 毕业项目不是“大一点的作业”

在这套课程里，毕业项目不是简单把功能做多，而是要展示:

- 选题判断
- 架构能力
- 系统边界意识
- 产品思维
- 复盘能力

## 2. 综合覆盖的含义

一个毕业项目至少应该覆盖:

- Agent 主线
- 知识或工具能力
- 状态或工作流能力
- 工程化层
- 前端体验层
- 交付与评测层

覆盖不一定都做到最复杂，但必须都能说清楚。

## 3. 毕业项目的价值标准

不是“功能多”，而是:

- 场景真实
- 用户明确
- 主路径清楚
- 架构有理由
- 风险被看见
- 下一版有方向

## 4. 答辩为什么重要

答辩的意义不是背答案，而是验证:

- 你能不能解释自己的设计
- 你能不能面对质疑
- 你能不能做出合理取舍

## 5. 复盘为什么是毕业的一部分

如果一个人只能展示结果，不能解释:

- 哪些地方做得对
- 哪些地方做得不好
- 下一版要怎么改

那说明掌握还不稳定。

## 6. 方法迁移才是真正掌握

Stage 8 最终要验证的不是“这个项目做出来了”，而是:

- 你是否能把同样的方法迁移到别的行业、别的产品、别的团队约束里

## 7. Stage 8 最重要的结论

真正毕业，不是完成一个项目，而是形成一套能够持续构建和迭代 Agent 产品的方法。
