# 01 Overview

## 学习目标

Stage 8 的目标，是把整套课程的知识和能力收束成一个真正的毕业项目闭环。

完成这一章后，学习者需要具备以下能力:

- 能独立定义一个 Agent 产品题目
- 能把后端、知识层、运行时、前端和评测整合起来
- 能给出一份有逻辑的产品与架构说明
- 能完成一次像真实团队评审那样的答辩与复盘

## 为什么 Stage 8 必须单独存在

如果没有这一阶段，前面的所有知识都可能停留在“我知道很多模块”。

但真正掌握的标志是:

- 能自己选题
- 能自己做取舍
- 能自己解释为什么这样设计
- 能接受评审并完成下一轮迭代规划

## 学习节奏建议

### Session 1

- 确定毕业项目方向
- 明确用户、场景和边界

### Session 2

- 完成综合架构设计
- 对照前 7 个阶段检查覆盖度

### Session 3

- 完成演示链路与交付说明
- 准备答辩材料

### Session 4

- 完成答辩与复盘
- 规划下一版本

## 本阶段的产出物

- 一份毕业项目说明
- 一张综合系统图
- 一份用户旅程说明
- 一份演示与交付说明
- 一份评测与复盘报告

## 毕业前的最后判断标准

如果学习者还不能回答下面 3 个问题，就说明还没有真正毕业:

- 为什么这个项目值得做
- 为什么这套架构适合它
- 如果继续做下一版，最优先该改什么
