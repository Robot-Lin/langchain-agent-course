# 01 Overview

## 本章目标

Stage 8 的目标，是把整套课程的知识和能力收束成一个真正的毕业项目闭环。

完成这一章后，学习者应该能做到：

- 独立定义一个 Agent 产品题目。
- 独立整合后端、知识层、运行时、前端和评测层。
- 用清楚的方式解释为什么这样设计。
- 像真实团队那样完成演示、答辩、复盘和下一版规划。

## 为什么 Stage 8 必须单独存在

如果没有这一章，前面的知识很容易停留在“我知道很多模块”。
但真正掌握的标志是：

- 你能自己选题。
- 你能自己做取舍。
- 你能自己解释设计逻辑。
- 你能接受评审并规划下一轮迭代。

## 学习节奏建议

### Session 1

- 确定毕业项目方向。
- 明确用户、场景和边界。

### Session 2

- 完成综合架构设计。
- 对照前 7 个阶段检查覆盖度。

### Session 3

- 完成演示链路与交付说明。
- 准备答辩材料。

### Session 4

- 完成答辩与复盘。
- 规划下一版路线。

## 本章产出物

学完 Stage 8，至少应该产出：

- 一份毕业项目说明。
- 一张综合系统图。
- 一份用户旅程与主路径说明。
- 一份演示与交付说明。
- 一份答辩问答准备。
- 一份复盘与下一版规划。

## 毕业前的最后判断标准

如果学习者还不能清楚回答下面三件事，就说明还没有真正毕业：

- 为什么这个项目值得做。
- 为什么这套架构适合它。
- 如果继续做下一版，最优先该改什么。
