# 04 Guided Project

本章引导项目不是固定单一题目，而是 `毕业项目框架引导`。学习者要从已有方向中选择一个，升级成完整毕业项目。

## 项目目标

选择一个自己最想做、也最能覆盖课程能力的 Agent 产品方向，完成从题目定义到综合设计、演示、答辩和复盘的完整过程。

## 为什么这一章不再给固定题目

因为毕业阶段要验证的是：

- 选题能力
- 取舍能力
- 综合能力

如果题目完全固定，反而很难验证这些能力。

## 项目最终交付物

完成这个项目时，至少要交付：

- 一份选题说明
- 一张综合架构图
- 一份用户旅程与主路径说明
- 一份演示脚本
- 一份答辩准备稿
- 一份复盘与下一版规划

## Step 1: 选择毕业方向

可选方向例如：

- 深度研究助手
- 学习教练 Agent
- 企业流程审批助手
- 内容工作台
- 个人操作系统型 Agent

## Step 2: 画综合架构

至少包括：

- 前端
- Agent runtime
- 知识或工具层
- 数据层
- 评测与追踪层
- 审批与治理层

## Step 3: 定义主路径与演示路径

明确：

- 用户如何进入
- 系统如何完成核心任务
- 演示时重点看什么
- 哪条路径最能代表产品价值

## Step 4: 准备答辩材料

至少准备：

- 选题说明
- 系统图
- 用户旅程
- 风险与边界
- 下一版规划

## Step 5: 做毕业复盘

项目做完后，必须回答：

- 哪些能力是真正掌握的
- 哪些能力仍需要补强
- 如果继续迭代，先做什么

## Codex 在这个项目里怎么配合

你可以让 Codex：

- 比较多个选题方向的覆盖度和风险
- 检查你的项目是否真的覆盖前 7 章能力
- 扮演答辩评审员追问你
- 帮你整理复盘框架

但不要让 Codex 直接替你决定项目方向或给出完整毕业答案。你真正要练的是独立取舍。
