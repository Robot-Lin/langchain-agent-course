# 06 Review Checklist

这一份文档用于 Stage 8 的毕业放行。不是确认你做完了项目，而是确认你是否真正掌握了整套课程的方法。

## 概念自测

如果下面任意 3 项还答不清楚，说明还不算真正毕业：

- 我能清楚说明我的选题价值。
- 我能画出综合系统图。
- 我能讲清用户旅程和主路径。
- 我能解释为什么选这条 Agent 主线。
- 我能说明前端、后端和 runtime 如何协同。
- 我能说明评测、追踪和治理设计。
- 我能完成一次完整答辩。
- 我能写出有价值的复盘和下一版规划。

## 综合判断自测

你应该已经能做到：

- 判断一个题目是否值得做。
- 判断一个方案是否真的覆盖课程核心能力。
- 判断一个项目哪些地方只是 Demo，哪些地方已经形成闭环。
- 判断下一版最该优先做什么。

## 表达与答辩自测

你应该已经能回答：

- 为什么这个项目值得做。
- 为什么它需要 Agent，而不是普通应用。
- 为什么你选择当前架构。
- 这个系统如何从前端到后端形成完整闭环。
- 这个系统如何被评测、追踪和治理。

## 复盘与迁移自测

你应该已经能写出：

- 哪些设计有效
- 哪些设计过度
- 哪些问题未解决
- 下一版规划
- 方法如何迁移到新场景

## 毕业答辩题

### 必答题

1. 这个项目为什么值得做。
2. 为什么它需要 Agent，而不是普通应用。
3. 为什么你选择当前架构。
4. 这个系统如何从前端到后端形成完整闭环。
5. 这个系统如何被评测、追踪和治理。

### 进阶题

1. 哪个设计是你最犹豫的取舍。
2. 如果用户规模扩大或团队缩小，架构会怎么改。
3. 如果再给你两周，你最优先做什么。
4. 如何把这套方法迁移到另一个行业。

## 毕业验收标准

通过 Stage 8，至少要满足：

- 完成 1 个完整毕业项目方案。
- 完成 1 份系统图和用户旅程说明。
- 完成 1 次完整答辩。
- 完成 1 份复盘报告。
- 完成 1 份下一版规划。

## 真正掌握的判断

如果学习者不只会展示当前项目，还能把方法迁移到新场景，就说明这套课程真正完成了它的目标。
