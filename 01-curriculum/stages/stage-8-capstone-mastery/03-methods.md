# 03 Methods

本文关注 Stage 8 的毕业项目推进方法。目标不是“把项目收尾”，而是完成一次完整的独立构建、表达、评审和迭代闭环。

## 方法一: 选题三问法

先问：

1. 这个项目为谁服务。
2. 这个场景为什么真的需要 Agent。
3. 这个题目能否覆盖课程核心能力。

如果这三问答不清，不建议立刻开做。

## 方法二: 覆盖检查法

在项目开始前，对照前 7 个阶段检查：

- 系统判断
- 高层 Agent
- 状态 / 工作流
- 复杂任务组织
- 工程化层
- 前端体验层
- 交付层

## 方法三: 主路径优先法

毕业项目也不要一开始就做“全能系统”。先定义：

- 一个核心用户
- 一个核心任务
- 一条可演示主路径

然后再补审批、评测、治理和体验细节。

## 方法四: 答辩准备法

答辩至少准备 5 类内容：

- 选题价值
- 用户与场景
- 架构图
- 演示路径
- 风险与下一版

## 方法五: 复盘四问法

项目完成后，至少回答：

1. 哪些设计是有效的。
2. 哪些设计是过度的。
3. 哪些问题还没解决。
4. 下一版最优先做什么。

## 方法六: 方法迁移法

最后再做一步：

- 用同一套方法快速说明另一个场景要怎么做。

这一步是检验是否真正掌握的关键。

## 方法目标

真正掌握 Stage 8，不是“项目收尾”，而是完成一次完整的独立构建、表达、评审和方法迁移闭环。

## 附录：Stage 8 读码与答辩对照表

这一节把毕业项目最容易讲空的部分，变成可以逐项检查的对照表。

### 文件解释层

- `项目总览文件`
  作用：说明项目目标、用户、主任务。
  答辩时要讲：为什么这个题目值得做。

- `前端主入口`
  作用：承接用户请求和界面主路径。
  答辩时要讲：用户第一步如何进入系统。

- `runtime 主入口`
  作用：组织 agent、graph、workflow。
  答辩时要讲：为什么这里是系统核心而不是某个工具文件。

- `工程化层文件`
  作用：观测、评测、治理、部署。
  答辩时要讲：为什么这部分决定系统能否长期演进。

### 主路径解释层

- `输入 -> runtime`
  要讲清：前端或 API 如何把请求送入系统。

- `runtime -> tools / knowledge / state`
  要讲清：系统如何做关键判断和能力调用。

- `runtime -> output / interrupt / stream`
  要讲清：中间结果和最终结果如何被用户看到。

- `旁路 -> trace / evaluation / audit`
  要讲清：为什么这些信息虽然不直接给用户看，但决定了产品是否可维护。

### 审核层

- `最脆弱文件`
  要讲清：为什么它脆弱，坏了会影响哪条主路径。

- `最值钱文件`
  要讲清：为什么它最能体现项目价值。

- `最容易过度设计的部分`
  要讲清：哪些功能看起来高级，但现在不值得做。

### 迁移层

- `同类场景迁移`
  要讲清：如果换一个行业或约束，哪些代码结构还能复用。

- `下一版路线`
  要讲清：下一步先改什么，以及为什么。

### 这一节最重要的原则

毕业项目的代码解释，不是逐文件背诵。真正有效的解释，是：

- 从目录到主路径
- 从主路径到关键文件
- 从关键文件到风险与取舍
- 从当前实现到下一版演进
