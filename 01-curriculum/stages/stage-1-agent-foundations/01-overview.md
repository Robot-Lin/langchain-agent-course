# 01 Overview

## 学习目标

Stage 1 的目标不是“会做项目”，而是建立后面所有项目都要依赖的底层判断。

完成这一章后，学习者需要具备以下能力:

- 能识别一个需求是否需要 Agent
- 能把需求拆成输入、推理、动作、状态、记忆、输出
- 能用自然语言说清一个 Agent 的系统结构
- 能说明系统的失败点、风险点和人工接管点

## 这一阶段为什么放在最前面

很多人接触 Agent 时，会自然地把重点放在提示词、模型和工具调用上。但真正的 Agent 设计问题，往往发生在更外层:

- 目标定义不清
- 工具边界不清
- 输出不可验证
- 没有状态设计
- 没有失败恢复
- 没有人机协作点

所以 Stage 1 的任务，是先把这些更本质的问题讲清楚。

## 学习节奏建议

### Session 1

- 读完核心概念
- 用自己的话解释“Agent 是系统，不是提示词”

### Session 2

- 学需求拆解方法
- 选择一个真实场景做最小 Agent 设计

### Session 3

- 跟做引导项目
- 让 Codex 帮你检查结构漏洞

### Session 4

- 完成独立练习
- 做阶段答辩或复盘

## 本阶段的产出物

- 一份最小 Agent 系统图
- 一份需求拆解表
- 一份失败点清单
- 一份人工介入点说明
- 一份自我复述记录

## 进入下一阶段前的判断标准

如果学习者还不能回答下面 3 个问题，就不建议进入 LangChain:

- 为什么这个系统需要 Agent，而不是普通应用逻辑
- 这个 Agent 需要什么外部能力
- 这个 Agent 最容易在哪些地方出错
