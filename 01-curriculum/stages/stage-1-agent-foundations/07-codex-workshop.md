# 07 Codex Workshop

这个文件不是“给 Codex 的说明”，而是“给学习者的协作手册”。它的目的不是让 Codex 更快替你完成，而是让你在和 Codex 的配合里，真正学会 `Stage 1`。

## Stage 1 与 Codex 配合的核心原则

在这一阶段，Codex 最适合扮演这些角色:

- 概念解释者
- 提问型教练
- 结构评审员
- 风险审查员
- 模拟答辩官

在这一阶段，Codex 最不适合扮演的角色是:

- 直接代做全部方案的人
- 一上来就给标准答案的人
- 用“更复杂的功能”掩盖设计不清的人

一句话原则:

> 在 Stage 1，Codex 应该帮你形成判断，而不是替你跳过判断。

## 使用前先告诉 Codex 什么

每次开始一个会话前，建议你至少告诉 Codex 这 5 件事:

1. 我现在处于 `Stage 1`。
2. 我的目标是学习判断，不是直接拿答案。
3. 请先问我问题，再帮我评审。
4. 不要直接进入代码实现。
5. 只按当前步骤推进，不要一次讲完全部。

一个通用开场模板:

```text
我现在在学习 LangChain Agent Course 的 Stage 1。
这一阶段我想训练的是 Agent 的系统判断，不是直接写代码。
请你先做导师和评审，不要直接给我完整答案。
每次只推进一步，先问我问题，再指出我的结构问题。
```

## 模式一: 概念讲解模式

适用场景:

- 你刚读完概念，还没真正理解
- 你能背定义，但讲不出自己的话
- 你想用例子理解状态、记忆、知识库的差别

推荐提问:

```text
请用课程新手能听懂的方式解释什么是 Agent。
不要只给定义，要用真实场景说明它和普通聊天机器人差在哪里。
```

```text
请用一个失败案例解释为什么状态和工具边界重要。
不要讲 API，重点讲系统为什么会失控。
```

```text
请比较状态、短期记忆、长期记忆、知识库。
每一项都给一个贴近课程场景的例子。
```

### 正确用法

- 让 Codex 解释以后，自己先复述
- 再让 Codex 指出你复述中的漏洞

### 错误用法

- 看完解释就继续下一章
- 不做复述，误以为“看懂了就算掌握”

## 模式二: 需求拆解模式

适用场景:

- 你有一个想法，但不知道怎么压缩成最小系统
- 你不确定这个场景到底应不应该做成 Agent
- 你想练最小 Agent 设计画布

推荐提问:

```text
我想做一个学习教练 Agent。
先不要写代码，请按用户、目标、输入、工具、状态、记忆、输出、风险、人工介入这 9 项带我拆解。
每一步先提问，不要直接帮我填答案。
```

```text
请先判断这个需求到底需不需要 Agent。
如果不需要，请明确告诉我更适合什么实现方式。
```

```text
请帮我把这个方案压缩成单一主任务。
你只负责指出哪里太散，不要替我发明更多功能。
```

### 正确用法

- 先自己答
- 再让 Codex 挑问题
- 最后自己改

### 错误用法

- 直接让 Codex 按 9 项全部填完
- 把 Codex 生成的结构当成自己的理解

## 模式三: 反方评审模式

适用场景:

- 你已经写出了一个初步方案
- 你想知道哪里最容易过度设计
- 你想提前发现失败路径和边界问题

推荐提问:

```text
站在严格评审角度，指出我这个 Agent 方案的 3 个最大问题。
优先关注目标是否过散、工具是否过多、状态和记忆是否混淆。
不要直接重写方案。
```

```text
请只从失败路径和风险边界角度审查我的方案。
告诉我哪里最容易失控，哪里最该加人工介入。
```

```text
请指出我这个设计里哪些部分其实根本不需要 Agent。
```

### 正确用法

- 欢迎被挑战
- 记录 Codex 指出的高频问题
- 根据反馈删功能，而不是继续加功能证明自己

### 错误用法

- 把评审当成否定
- 每次被指出问题，就继续堆更多解释和功能

## 模式四: 最小可行压缩模式

适用场景:

- 你发现自己的方案越来越大
- 你开始想加搜索、记忆、多 Agent、审批流、前端全套
- 你已经闻到“过度设计”的味道了

推荐提问:

```text
请把这个复杂想法压缩成一个最小可行 Agent。
只保留一条主路径、一个主任务、一个最必要工具。
同时告诉我删掉了什么，以及为什么删。
```

```text
请从 Stage 1 视角判断，我这个方案哪些部分应该放到后续阶段，而不是现在就做。
```

### 正确用法

- 接受删减
- 先保护主路径
- 把“以后再做”视为成熟，而不是遗憾

### 错误用法

- 口头上说“做减法”，实际还是想把所有能力留住

## 模式五: 模拟答辩模式

适用场景:

- 你准备做 Stage 1 放行检查
- 你想测试自己是不是已经能脱稿讲清楚
- 你希望进入 Stage 2 前做一次口头压力测试

推荐提问:

```text
你来当导师，逐题提问我 Stage 1 的答辩题。
不要先给答案，等我回答完再指出哪里不清楚。
```

```text
请模拟一个严格但友好的答辩官。
每次只问一个问题，并追问我为什么。
```

### 正确用法

- 先脱稿回答
- 再让 Codex 追问
- 最后自己写复盘

### 错误用法

- 让 Codex 先给标准答案再背

## 模式六: 项目带学模式

适用场景:

- 你在做 [04-guided-project.md](./04-guided-project.md)
- 你在做 [05-challenges.md](./05-challenges.md)
- 你准备进入课后项目

推荐提问:

```text
请按 Stage 1 的知识点顺序带我完成 FAQ Agent。
每次只推进一步，并先告诉我这一步对应哪个概念和方法。
不要一次给完整答案。
```

```text
我现在做的是 Stage 1 课后项目。
请不要代做，只按验收标准提问、审查和评分。
```

### 正确用法

- 用 Codex 做带学教练
- 每一步都先解释为什么
- 保留你自己的初版答案和修改痕迹

### 错误用法

- 跳过自己的回答，直接复制 Codex 的方案

## 推荐学习动作

一个很稳的对练闭环是:

1. 先让 Codex 解释。
2. 再自己复述。
3. 再让 Codex 挑错。
4. 再修正你的表达。
5. 最后让 Codex 模拟评审或答辩。

这个顺序很重要，因为它会强迫你从“被动理解”进入“主动表达”。

## 什么时候不要让 Codex 直接回答

在下面这些场景里，最好不要让 Codex 直接给完整答案:

- 你正在做 challenge
- 你正在做课后项目
- 你准备做答辩
- 你还没有自己写过第一版方案

更好的做法是:

- 先写你的版本
- 再让 Codex 做批改
- 最后自己再改一版

## 一个完整的 Stage 1 会话模板

你可以直接复制下面这段开始一轮高质量学习:

```text
我现在在学习 LangChain Agent Course 的 Stage 1。
我希望你扮演导师和评审，而不是直接代做。
请先根据我给你的场景，判断它是否真的需要 Agent。
如果需要，再按用户、目标、输入、工具、状态、记忆、输出、风险、人工介入这几个维度，一步一步带我拆解。
每次只推进一步。
先问我问题，再指出我回答里的结构问题。
不要直接进入代码实现，也不要一次给完整答案。
```

## Stage 1 结束前，你应该能做到的协作状态

如果这一章学得比较扎实，到了最后你和 Codex 的关系应该是:

- 你负责提出初步判断
- Codex 负责挑战你的判断
- 你负责修正结构
- Codex 负责指出风险和漏洞
- 你最终能不依赖 Codex，也把一个最小 Agent 讲清楚

这才说明 Codex 真正起到了“放大学习效率”的作用，而不是替你绕过了学习过程。
