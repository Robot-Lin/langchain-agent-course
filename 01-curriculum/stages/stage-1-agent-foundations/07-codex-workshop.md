# 07 Codex Workshop

这一部分是给学习者直接复制使用的 Codex 对练脚本。

## 模式一: 概念讲解

可以对 Codex 说:

- 用课程新手能听懂的方式解释什么是 Agent
- 用真实项目比较 Agent 和普通聊天机器人
- 用一个失败案例解释为什么状态和工具边界重要

## 模式二: 需求拆解

可以对 Codex 说:

- 我想做一个学习教练 Agent，先别写代码，帮我拆成最小系统
- 按用户、目标、输入、工具、状态、记忆、输出、风险、人工介入点这 9 项帮我整理
- 帮我判断这个需求哪里设计过度了

## 模式三: 反方评审

可以对 Codex 说:

- 站在严格评审角度，指出我的 Agent 设计问题
- 哪些部分其实不需要 Agent
- 哪些地方最可能失控

## 模式四: 最小可行压缩

可以对 Codex 说:

- 把这个复杂想法压缩成一个最小可行 Agent
- 只保留一条主路径，并告诉我删掉了什么、为什么

## 模式五: 阶段答辩陪练

可以对 Codex 说:

- 你来当导师，逐题提问我 Stage 1 的答辩题
- 不要直接给答案，先等我回答，再指出我哪里讲得不清楚

## 推荐学习动作

1. 先让 Codex 解释
2. 再自己复述
3. 再让 Codex 挑错
4. 再修正你的表达
5. 最后让 Codex 模拟评审

## 使用提醒

- 不要一开始就让 Codex 直接给完整方案
- 先让它帮你拆，再让它帮你评
- 你的目标不是尽快拿到答案，而是尽快形成自己的结构化理解
