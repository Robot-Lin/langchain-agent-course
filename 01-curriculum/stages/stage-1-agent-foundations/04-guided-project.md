# 04 Guided Project

本章的引导项目是 FAQ Agent。这一章的重点不是把项目做得最大，而是把本章知识点真实落到功能、文件、代码和审核判断上。

## 项目目标

把一个面向课程新手的 FAQ Agent 做小、做清楚、做出边界。

## 对应知识点

需求压缩、工具必要性判断、状态与记忆区分、澄清逻辑、人工介入边界

## 知识点如何落到代码上

重点映射到这些实现层：

- 问题分类逻辑、课程资料读取入口、澄清判断、最小状态对象
- 主路径输入如何进入系统
- 关键状态、关键判断或关键输出在哪里形成

## 推荐推进顺序

1. 先定义唯一主任务与不做什么
2. 再明确输入、输出与有效结果标准
3. 判断最小工具集和最小状态
4. 设计澄清点、失败点和人工介入点
5. 最后整理成最小实现草图并做代码导读

## 本章必须完成的代码导读

做完项目设计或实现后，至少补做下面这轮读码：

- 先列出关键文件或关键模块
- 再说清它们各自负责什么
- 再解释一条主路径如何从输入走到输出
- 最后指出如果你来审核，会优先看哪 2 到 3 个风险点

## 最终交付物

- 项目目标说明、知识点到功能映射、功能到文件映射、关键逻辑解释
- 一份主路径解释
- 一份用户审核要点清单

## 推荐给 Codex 的说法

- 请带我按文件解释这个 FAQ Agent 的主路径，不要先改代码。
- 不要只告诉我结果，先告诉我每个知识点落到了哪些文件和逻辑上。
- 如果我讲不清楚代码逻辑，不要直接进入下一步。
