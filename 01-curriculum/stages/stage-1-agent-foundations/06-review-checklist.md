# 06 Review Checklist

## 自测清单

如果下面任意 3 项做不到，建议不要急着进入 Stage 2。

- 我能用自己的话解释 Agent 是什么
- 我能区分 Agent、聊天机器人、工作流
- 我能说明工具在系统中的作用
- 我能区分状态、记忆和知识库
- 我能说出至少 3 类失败
- 我能解释为什么某些动作必须人工确认
- 我能把一个模糊需求压缩成最小系统

## 阶段答辩题

### 必答题

1. 如果一个系统只需要回答问题，为什么不一定需要 Agent
2. 如果一个系统要调用搜索、数据库和发消息功能，它的风险点在哪里
3. 如果一个系统跨多轮任务工作，为什么需要状态
4. 为什么长期记忆不应该等于“把所有对话都存起来”

### 进阶题

1. 一个“看起来很聪明”的 Agent，为什么仍可能不适合上线
2. 为什么一个项目应该先设计人工介入点，而不是最后再补
3. 为什么最小可行版本对 Agent 特别重要

## 放行标准

学习者至少要做到:

- 完成 1 个最小 Agent 画布
- 完成 1 个引导项目拆解
- 完成 1 组挑战题
- 完成 1 次口头或书面答辩

## 进入 Stage 2 前的最后确认

只有当学习者已经具备“先做结构判断，再做框架实现”的意识时，Stage 2 才会真正有效。
