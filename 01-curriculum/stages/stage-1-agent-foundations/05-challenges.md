# 05 Challenges

这一章的练习不只是为了检查有没有做出东西，更是为了检查你能不能看懂它为什么这样实现。

## 必做挑战

- 给一个课程 FAQ Agent 写出最小文件职责表
- 解释“问题分类 -> 读取课程信息 -> 返回回答或澄清”的主路径
- 指出这里为什么通常不需要长期记忆或高风险工具
## 提交要求

每道题尽量都包含：

- 一份自然语言说明
- 一份知识点到功能的映射
- 一份功能到文件或模块的映射
- 至少 1 段关键逻辑解释

## 额外读码挑战

除了原本练习，还要额外完成一件事：

- 选一条主路径，按输入、处理中间步骤、输出三个阶段讲清楚
- 回答时要明确说出文件、函数、组件、节点或模块职责
- 最后再指出 1 个你认为值得继续改进的实现点

## 如何让 Codex 参与

- 先让 Codex 只做提问和检查，不直接给完整答案
- 如果已经有代码，就先让 Codex 带你读，再让它提改进建议
- 最后按“功能完成度 + 代码理解度”一起评审
