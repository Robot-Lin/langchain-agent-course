# 05 Challenges

这一部分要求学习者开始自己完成，不再只跟着看。

## Challenge 1: 判断题型练习

请分别判断下面需求是否真的需要 Agent，并给出理由。

### 题目 A

用户上传一段文本，系统输出固定 JSON 结构。

### 题目 B

用户希望系统自动查找资料、整理答案、必要时追问、最后给出带来源的总结。

### 题目 C

用户点击按钮后，系统把表单内容发给后台保存。

### 题目 D

用户希望系统根据任务状态决定是否继续下一步、是否申请人工确认、是否调用外部系统。

## Challenge 2: 需求拆解练习

从下面 3 个场景中选 1 个，完成最小 Agent 画布。

- 学习教练
- 个人日程助理
- 研究资料整理助手

要求至少写清:

- 用户
- 目标
- 输入
- 工具
- 状态
- 记忆
- 输出
- 风险
- 人工介入点

## Challenge 3: 缩小复杂度练习

把下面这个想法压缩成最小可行版本:

“做一个全能个人助手，能查资料、写邮件、管日程、读文件、记住偏好、自动执行工作流。”

要求:

- 只保留一个主任务
- 只保留一个用户类型
- 只保留一个最必要工具
- 写出为什么删掉其他能力

## Challenge 4: 失败复盘练习

假设一个 Agent 出现了下面情况:

- 回答很流畅，但引用了错误知识
- 自动调用了不该调用的工具
- 明明没完成任务，却说已经完成

请判断这分别属于什么类型的失败，并写出至少一个改进思路。

## Challenge 5: 口头答辩练习

请尝试脱稿回答:

- 什么是 Agent
- Agent 和聊天机器人的差别是什么
- 什么情况下必须引入工具
- 什么情况下必须引入状态
- 记忆和知识库为什么不是一回事
- 为什么高风险动作要设置人工确认
