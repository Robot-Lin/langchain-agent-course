# 08 Session Roadmap

这个文件把 `Stage 1` 拆成一组真正可以执行的学习 session。你可以自己按节奏推进，也可以让 Codex 按这个顺序一节一节带你学。

## 建议总时长

- 轻量模式: 3 到 4 天
- 标准模式: 5 到 7 天
- 深挖模式: 1 到 2 周

## Session 1: Agent 是系统，不是提示词

### 学习目标

- 建立对 Agent 的系统级理解
- 区分 Agent 与普通聊天机器人的差别
- 建立 “不是所有需求都需要 Agent” 的判断意识

### 先读什么

- [01-overview.md](./01-overview.md)
- [02-concepts.md](./02-concepts.md)

### 要做什么

- 用自己的话解释什么是 Agent
- 写出 3 个“不需要做成 Agent”的需求
- 写出 3 个“更像 Agent”的需求

### 交付物

- 一段 100 到 150 字的理解复述
- 一张简单的 “聊天机器人 vs Agent” 对比表

### 推荐对 Codex 说

- 不要给我标准答案，先听我解释什么是 Agent，再指出我哪里还像在讲提示词而不是系统。
- 帮我判断下面这 6 个场景，哪些值得做成 Agent，哪些不值得。

## Session 2: 工具、状态、记忆、知识库、审批

### 学习目标

- 分清不同系统能力的职责
- 避免把状态、记忆、知识库混成一团
- 初步建立审批和人工介入意识

### 先读什么

- [02-concepts.md](./02-concepts.md)
- [09-case-comparisons.md](./09-case-comparisons.md)

### 要做什么

- 选 3 个案例，逐一判断是否需要工具、状态、记忆、审批
- 写出你自己的区分标准

### 交付物

- 一张能力判断表
- 一份 5 条以内的判断原则

### 推荐对 Codex 说

- 我会给你一个场景，你只负责追问我为什么需要工具、为什么需要状态，不要直接帮我决定。
- 帮我检查我是否把知识库误写成记忆了。

## Session 3: 最小 Agent 设计画布

### 学习目标

- 学会把模糊需求压缩成单一目标系统
- 学会先做减法，再做加法
- 能写出最小可行 Agent 设计画布

### 先读什么

- [03-methods.md](./03-methods.md)

### 要做什么

- 选一个你自己的 Agent 想法
- 填写最小 Agent 设计画布
- 删除至少 3 个你一开始想加、后来确认暂时不该加的功能

### 交付物

- 一份最小 Agent 设计画布
- 一份 “现在不做什么” 的减法清单

### 推荐对 Codex 说

- 请你像设计评审一样，逼我把目标缩成单一主任务。
- 帮我指出这个方案哪里有过度设计。

## Session 4: 失败路径与审批门

### 学习目标

- 在实现前就能写出失败点
- 学会识别高风险动作
- 能设计最基础的人机协作边界

### 先读什么

- [03-methods.md](./03-methods.md)
- [06-review-checklist.md](./06-review-checklist.md)

### 要做什么

- 列出至少 5 个失败点
- 列出至少 3 个应该暂停给人确认的地方
- 说明每个审批门为什么存在

### 交付物

- 一份失败路径清单
- 一份审批门说明

### 推荐对 Codex 说

- 不要替我补充功能，只从失败角度审查我的方案。
- 站在用户视角告诉我，最容易先抱怨什么。

## Session 5: 跟做 FAQ Agent

### 学习目标

- 把前 4 个 session 的判断能力落到一个具体项目上
- 通过跟学项目建立最小 Agent 设计手感

### 先读什么

- [04-guided-project.md](./04-guided-project.md)
- [Stage 1 跟学项目](../../../03-projects/stage-1/guided-project-faq-agent.md)

### 要做什么

- 跟着项目一步步做系统拆解
- 每一步先解释为什么这样设计，再继续下一步

### 交付物

- FAQ Agent 自然语言架构说明
- 最小系统图
- 风险与澄清策略

### 推荐对 Codex 说

- 先按 Stage 1 的知识点顺序带我做 FAQ Agent，每次只推进一步。
- 在每一步开始前，先告诉我这一步对应的是哪个概念和方法。

## Session 6: 独立完成课后项目

### 学习目标

- 不靠标准答案完成独立设计
- 接受基于标准的评审，而不是只看“像不像”

### 先读什么

- [05-challenges.md](./05-challenges.md)
- [Stage 1 课后项目](../../../03-projects/stage-1/homework-project-learning-coach.md)

### 要做什么

- 先独立写方案
- 再让 Codex 按验收标准评分和指出缺口

### 交付物

- 学习教练 Agent 方案
- 自评结果
- Codex 评审反馈摘要

### 推荐对 Codex 说

- 这是我的 Stage 1 课后项目，请不要直接给完整答案，只按验收标准给我反馈。
- 先给我打分，再指出最值得优先修的 3 个问题。

## Session 7: 复盘与阶段放行

### 学习目标

- 把判断经验内化成方法
- 确认是否真的准备进入 `LangChain`

### 先读什么

- [06-review-checklist.md](./06-review-checklist.md)
- [07-codex-workshop.md](./07-codex-workshop.md)

### 要做什么

- 回答 Stage 1 的放行问题
- 用自己的案例重新解释状态、记忆、知识库的区别
- 说明为什么你的系统需要或不需要 Agent

### 放行标准

你至少应该能稳定做到:

- 不把提示词当成系统设计本身
- 不把状态、记忆、知识库混在一起
- 不为“看起来高级”而盲目加工具和多 Agent
- 能清楚说出系统何时应该暂停交给人

## 建议学习循环

每完成一个 session，都做一次同样的闭环:

1. 复述这节最核心的一句话。
2. 写下你删掉了什么复杂度。
3. 写下你新增了什么边界意识。
4. 让 Codex 只从审查角度给你反馈。

如果你希望真正按节奏开始学，现在最适合从 `Session 1` 正式推进。
