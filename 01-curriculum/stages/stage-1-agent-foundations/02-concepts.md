# 02 Concepts

## 1. Agent 是什么

在这套课程里，Agent 指的是一个能够围绕目标工作、使用上下文、调用外部能力、在一定流程中完成任务的系统。

它通常包括:

- 模型
- 系统指令
- 用户输入
- 上下文
- 工具
- 状态
- 记忆
- 输出结构
- 反馈回路

关键理解:

- 模型负责生成，不等于系统负责完成任务
- Agent 的价值来自“能在系统环境中行动”
- 设计 Agent 本质上是设计任务完成机制

## 2. Agent 和普通聊天机器人的差别

普通聊天机器人的重点通常是“更好地回答”。

Agent 的重点通常是:

- 理解目标
- 选择动作
- 使用工具
- 管理状态
- 在必要时暂停、询问或交还给人

一个简单判断方法:

- 如果系统只需要回答问题，且不需要调用外部能力、维护状态或执行流程，通常不必急着做成 Agent。

## 3. 什么是工具

工具是 Agent 与外部世界连接的方式。

工具可以是:

- 搜索
- 数据库查询
- 文件读写
- 浏览器操作
- 发送消息
- 调用业务 API

引入工具的原因:

- 模型本身不知道实时信息
- 模型不能直接执行外部动作
- 模型输出需要被外部系统消费

需要警惕:

- 工具越多，不代表系统越强
- 错误工具边界会导致系统不可控

## 4. 什么是状态

状态是系统在执行过程中需要持续保存的信息。

常见状态包括:

- 当前任务阶段
- 已完成步骤
- 用户确认结果
- 当前计划
- 中间输出

没有状态时，系统往往只能“每次重新来过”。

## 5. 什么是记忆

记忆不是“所有历史都塞进去”，而是系统对有价值信息的保留与调用。

基础区分:

- 短期记忆: 当前会话或当前任务上下文
- 长期记忆: 跨会话保留的用户偏好、事实、经验
- 知识库: 外部文档和资料，不等于记忆

根据 LangChain 官方 memory 文档，短期上下文和长期数据在系统中需要分层处理，而不是混成一团。

参考:

- [Memory overview](https://docs.langchain.com/oss/python/concepts/memory)

## 6. 什么是 human-in-the-loop

不是所有动作都应该自动执行。

当 Agent 涉及以下操作时，通常需要人类审核:

- 高风险操作
- 不可逆操作
- 费用敏感操作
- 对外发送行为
- 关键信息修改

官方文档中也明确把 human-in-the-loop 视为工具调用和工作流中的重要能力，而不是可有可无的“锦上添花”。

参考:

- [LangChain human-in-the-loop](https://docs.langchain.com/oss/python/langchain/human-in-the-loop)
- [LangChain frontend human-in-the-loop](https://docs.langchain.com/oss/python/langchain/frontend/human-in-the-loop)

## 7. 什么是 Agent 的失败

Agent 的失败不只是“答案错了”。

还包括:

- 任务边界错了
- 调错工具
- 漏掉关键步骤
- 输出格式不可用
- 误以为任务完成了
- 没有停在应该请求确认的地方
- 用户不知道系统当前在做什么

## 8. 新手最容易犯的 8 个错误

1. 先堆功能，不先定目标
2. 把提示词当全部系统设计
3. 用工具掩盖需求不清
4. 不设计输出格式
5. 不区分状态、记忆、知识库
6. 不给高风险动作设审批
7. 只看演示成功，不看失败路径
8. 把“能跑一次”当成“可以长期用”

## 9. Stage 1 最重要的结论

在进入 LangChain、LangGraph、Deep Agents 之前，先学会这句话:

“一个 Agent 项目首先是一个系统设计问题，其次才是框架实现问题。”
