# 05 Challenges

这一部分开始要求学习者不只是“理解 LangGraph 概念”，而是主动设计图、设计 state、设计 interrupt、设计恢复路径，并判断为什么这里必须使用 LangGraph。

建议按顺序做，不要跳题。因为 `Stage 3` 真正训练的是一个渐进过程:

- 先判断为什么需要图
- 再设计 state
- 再设计 interrupt 与恢复
- 再复盘边界
- 最后进入独立项目

## Challenge 1: 为什么需要 LangGraph

### 目标

训练你判断: 哪些项目已经明显进入图系统问题，而不是高层原型问题。

### 任务

请判断下面哪些设计已经明显需要 LangGraph。

### 题目 A

单工具研究助理，输出结构化摘要。

### 题目 B

系统需要多阶段审批、暂停等待、失败恢复和继续执行。

### 题目 C

结构化抽取 Agent，输入文本，输出固定 JSON。

### 题目 D

多阶段研究流程，需要在关键节点等待人工调整方向，并在失败后恢复继续。

### 你至少要回答

- 当前更适合 LangChain 还是 LangGraph
- 为什么
- 最关键的判断依据是什么

## Challenge 2: state 设计练习

### 目标

训练你用最小集方式设计 state。

### 任务

请选择一个工作流场景，为它设计 state:

- 审批流程
- 多阶段研究流程
- 日程协调流程

### 你至少要写清

- 字段名
- 每个字段为什么需要
- 哪些字段影响后续路径
- 哪些字段支持恢复
- 哪些字段支持审批或回显

### 合格标准

- state 不是随手堆字段
- 能解释每个字段的存在理由
- 能看出它与流程控制强相关

## Challenge 3: interrupt 设计练习

### 目标

训练你把暂停设计成正式运行时能力。

### 任务

请为一个工作流场景设计至少 2 个 interrupt 点，并说明:

- 为什么这里要停
- 谁来决定继续
- 拒绝或修改后走哪条分支

### 合格标准

- interrupt 不是普通聊天追问
- interrupt 位置有业务意义
- 停下来以后能说明如何恢复

## Challenge 4: 恢复路径练习

### 目标

训练你把失败变成图里可处理的状态。

### 任务

针对下面 3 种情况，为一个工作流设计恢复路径:

- 审批被拒绝
- 工具执行失败
- 用户补充了缺失信息

### 你至少要回答

- 恢复到哪一步
- 为什么恢复到这里
- 为什么不是从头重来

### 合格标准

- 恢复逻辑具体
- 能体现状态连续性
- 不把“重试一次”当成全部恢复设计

## Challenge 5: 图式复盘练习

### 目标

训练你用图系统视角复盘问题，而不是只看模型表现。

### 任务

假设一个图系统出现下面这些问题，请判断它更像是哪类设计问题:

- 审批后系统不知道从哪里继续
- 用户看到系统一直在运行，但不知道现在处于哪个阶段
- 失败后只能从头开始
- state 越存越多，团队自己都解释不清

### 你至少要回答

- 这是 state、interrupt、恢复还是 streaming / 可见性问题
- 为什么会发生
- 系统层应该怎么改

## Challenge 6: Stage 2 升级练习

### 目标

训练你把 `Stage 2` 的高层原型推进成 `Stage 3` 的图系统设计。

### 任务

请选择一个 Stage 2 项目，把它升级成 LangGraph 设计，并至少补上:

- 工作流主路径
- state 字段
- interrupt 点
- 恢复路径
- 为什么现在必须使用 LangGraph

### 合格标准

- 能看出从高层原型到图系统的升级
- 不是简单换个框架名
- 新增内容真正围绕运行时能力展开

## Challenge 7: 综合小作业

### 目标

把前面练习过的能力拼起来，完成一个完整的图系统说明。

### 任务

任选一个你熟悉的流程场景，写出一份 `250-500` 字的 LangGraph 工作流说明，并附带:

- 一张文字版工作流图
- 一份 state 清单
- 至少 2 个 interrupt 点
- 至少 2 条恢复路径
- 至少 5 个失败点
- 一段“为什么必须使用 LangGraph”的边界说明

## 自评 Rubric

做完这一章后，你可以按下面 5 个维度给自己打分，每项 `1-5` 分:

- 图式理解: 我能否解释为什么这里需要图
- state 设计: 我能否设计最小但够用的 state
- interrupt 设计: 我能否把暂停设计进系统
- 恢复意识: 我能否让失败后继续，而不是重来
- 边界判断: 我能否说明为什么这里必须使用 LangGraph

### 粗略参考

- `20-25` 分: 可以进入课后项目或准备进入 Stage 4
- `15-19` 分: 建议再做一轮挑战题
- `14` 分及以下: 建议回看 `02-concepts.md` 和 `03-methods.md`

## 推荐学习闭环

每完成一个 challenge，都做一次小闭环:

1. 写下你的答案。
2. 让 Codex 只做图系统评审。
3. 记录它指出的关键问题。
4. 用你自己的语言修正答案。
5. 总结这题帮你建立了什么运行时判断标准。
