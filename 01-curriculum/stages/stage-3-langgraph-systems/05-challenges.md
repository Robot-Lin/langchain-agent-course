# 05 Challenges

## Challenge 1: 升级判断

请判断下面哪些项目已经明显应该用 LangGraph 思维。

### 题目 A

单工具研究助手，只输出结构化摘要。

### 题目 B

用户提交任务后，系统要生成计划、请求审批、执行动作，并在失败时返回重新规划。

### 题目 C

固定字段抽取器。

### 题目 D

需要暂停等待用户补充输入，再继续执行后续节点。

## Challenge 2: State 设计练习

请为一个“审批流 Agent”设计 state。

要求:

- 至少 6 个字段
- 每个字段都写出保留理由
- 至少指出 2 个不应该放进 state 的信息

## Challenge 3: Interrupt 设计练习

请为一个任务型 Agent 设计至少 3 个 interrupt 点，并说明:

- 为什么在这里停
- 如果不停会有什么风险

## Challenge 4: 恢复路径练习

假设系统出现下面问题:

- 审批被拒绝
- 工具调用失败
- 用户补充信息后想继续原任务

请为每种情况写出恢复策略。

## Challenge 5: 图式复盘练习

看一个你在 Stage 2 做过的项目，回答:

- 如果升级到 LangGraph，哪些节点会出现
- 哪些状态必须显式化
- 哪些地方应该设置 interrupt
