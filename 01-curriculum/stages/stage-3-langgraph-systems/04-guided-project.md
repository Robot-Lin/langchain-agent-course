# 04 Guided Project

本章的引导项目是 带人工审批的流程 Agent。这一章的重点不是把项目做得最大，而是把本章知识点真实落到功能、文件、代码、跨文件连接和审核判断上。

## 项目目标

把一个需要审批、恢复和分支的任务做成可解释的 LangGraph 工作流。

## 对应知识点

state、branching、interrupt、durable execution、恢复路径、审批设计

## 知识点如何落到代码上

重点映射到这些实现层：

- state schema、node handlers、router、interrupt 点、resume 或 retry 路径
- 主路径输入如何进入系统
- 关键状态、关键判断或关键输出在哪里形成
- 不同文件之间如何把一张图和一条线程串起来

## 推荐推进顺序

1. 先画主路径和关键分支
2. 再定义最小 state 字段
3. 设计 interrupt 与审批点
4. 补恢复路径和失败处理
5. 先对照 `02-concepts.md` 后半部分的代码教材和 `03-methods.md` 末尾的 API 对照附录理解最小图结构
6. 完成设计后先按节点和边逐段读图执行代码
7. 再做一次整图和全项目扫描，解释每个主要文件、节点和运行时能力的连接关系

## 本章必须完成的代码导读

做完项目设计或实现后，至少补做下面这轮读码：

- 先列出关键文件或关键模块
- 再说清它们各自负责什么
- 再解释一条主路径如何从输入走到输出
- 再按目录把主要文件扫一遍，说明哪些是 state 层、哪些是 node 层、哪些是 router / runtime 层
- 最后指出如果你来审核，会优先看哪 2 到 3 个风险点

## 最终交付物

- 工作流图、state 说明、节点职责表、关键分支解释
- 一份主路径解释
- 一份全项目文件地图
- 一份用户审核要点清单

## 推荐给 Codex 的说法

- 请按 state、node、router 三层带我读懂这张 LangGraph 图。
- 不要只告诉我结果，先告诉我每个知识点落到了哪些文件和逻辑上。
- 在主路径讲完之后，再带我按文件把整个项目扫一遍。
- 如果我讲不清楚代码逻辑，不要直接进入下一步。
