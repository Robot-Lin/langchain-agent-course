# 06 Review Checklist

## 自测清单

如果下面任意 3 项做不到，建议不要急着进入 Stage 4。

- 我能解释为什么这里需要图
- 我能设计一个最小 state
- 我能设计分支和失败路径
- 我能解释 durable execution 的实际意义
- 我能说明 interrupt 与普通追问的差别
- 我能说明审批与恢复如何联动
- 我能做一个 LangGraph 工作流设计
- 我能解释为什么这里不适合只停留在 LangChain 高层

## 阶段答辩题

### 必答题

1. 为什么状态建模是系统设计问题，而不只是实现细节
2. 为什么 interrupt 不能简单理解成“多问一句用户”
3. durable execution 对真实业务流程的价值是什么
4. 为什么一个带审批的流程特别适合用 LangGraph

### 进阶题

1. 如果 state 设计过重，会带来什么问题
2. streaming 为什么和 Agent 可信体验相关
3. 你会如何把 Stage 2 的研究助理升级成 LangGraph 系统

## 放行标准

学习者至少要做到:

- 完成 1 个 LangGraph 工作流设计
- 完成 1 个跟学项目
- 完成 1 个课后独立项目
- 完成 1 次围绕 state、interrupt、恢复策略的答辩或评审

## 进入 Stage 4 前的最后确认

只有当学习者已经能把复杂任务表达成状态化系统，而不是只会堆节点，进入 Deep Agents 才会真正有意义。
