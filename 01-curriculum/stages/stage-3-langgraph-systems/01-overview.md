# 01 Overview

## 学习目标

Stage 3 的目标，是让学习者从“会搭一个 Agent 原型”升级到“会设计一个可控、可恢复、可审批的 Agent 系统”。

完成这一章后，学习者需要具备以下能力:

- 能把一个任务拆成图结构
- 能为工作流定义 state
- 能设计分支、暂停、恢复和审批机制
- 能说明为什么这个系统必须使用 LangGraph

## 为什么这一阶段紧接 Stage 2

Stage 2 适合做高层原型，但当系统开始出现下面情况时，就会逼近它的边界:

- 多阶段流程
- 明显分支
- 长任务执行
- 失败恢复
- 审批与人工接管

这时继续强行用高层封装，通常会让系统越来越难解释、难维护。

## 学习节奏建议

### Session 1

- 读完核心概念
- 理解为什么图是这里的主角

### Session 2

- 学 state 设计和控制流方法
- 把一个 Stage 2 原型升级成图式设计

### Session 3

- 跟做引导项目
- 重点练审批和 interrupt

### Session 4

- 完成课后独立项目
- 让 Codex 按标准做架构评审

## 本阶段的产出物

- 一份 LangGraph 工作流图
- 一份 state 设计说明
- 一份审批与恢复策略
- 一份为什么必须使用 LangGraph 的说明

## 进入下一阶段前的判断标准

如果学习者还不能回答下面 3 个问题，就不建议进入 Deep Agents:

- 哪些问题是 state 设计问题，而不是 prompt 问题
- interrupt 在这个项目里真正暂停了什么
- 为什么这个系统需要恢复能力
