# Stage 6: Agent Frontend

## 阶段目标

- 把 Agent 变成真正可用的交互产品
- 学会设计用户可理解、可控制、可审批的 Agent 体验
- 建立前端与 Agent runtime 的协同认知

## 关联官方主线

- [LangChain frontend structured output](https://docs.langchain.com/oss/python/langchain/frontend/structured-output)
- [LangChain frontend human-in-the-loop](https://docs.langchain.com/oss/python/langchain/frontend/human-in-the-loop)
- [LangGraph streaming](https://docs.langchain.com/oss/python/langgraph/streaming)

## 学习重点

- Chat UX 与 Agent UX
- 流式输出
- 任务计划可视化
- 工具执行状态展示
- 审批界面
- 历史记录与记忆可见性
- 失败反馈与解释性

## 阅读知识

- 为什么 Agent 产品不能只有一个聊天框
- 为什么 interrupt、streaming 和审批设计会直接影响体验
- 为什么用户必须看见 Agent 当前状态

## 方法掌握

- Agent UX 设计法:
  先设计主任务流
  再设计状态展示
  再设计人机接管点
  再设计失败反馈

- 可控性交互法:
  重要动作先预览
  高风险动作先审批
  长任务过程持续可见

## 引导项目

- 研究控制台前端
- 带审批的企业助手界面
- 多面板创作工作台

## 主动练习

- 为一个 Agent 项目画出前端布局
- 标出:
  用户输入区
  计划区
  工具状态区
  审批区
  历史记录区

## 放行标准

- 能解释 Agent UX 与聊天 UX 的差别
- 能设计一个包含状态与审批的界面
- 能说清前端如何提升信任感和可控性
