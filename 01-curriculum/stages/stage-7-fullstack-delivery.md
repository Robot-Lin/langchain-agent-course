# Stage 7: Full-stack Delivery

## 阶段目标

- 将 Agent runtime、数据层、知识层和前端连接成一个完整产品
- 学会部署、观测、评测和迭代
- 具备做全栈 Agent 项目的交付能力

## 关联官方主线

- [LangSmith Deployment](https://docs.langchain.com/langsmith/deployment)
- [Agent Server API reference](https://docs.langchain.com/langsmith/server-api-ref)
- [LangSmith Observability](https://docs.langchain.com/langsmith/observability)
- [LangSmith Evaluation](https://docs.langchain.com/langsmith/evaluation)

## 学习重点

- Full-stack architecture
- 前后端通信
- Agent Server
- Streaming UI integration
- Deployment
- Observability
- Evaluation
- 审计、权限和产品边界
- 框架选型与迁移

## 阅读知识

- 一个 Agent 产品从请求到结果的完整链路
- 为什么部署、评测和观测要在产品视角里理解
- 为什么同一个需求在不同框架下会有不同最优架构

## 方法掌握

- 全栈组装法:
  先定用户旅程
  再定前端界面
  再定 Agent runtime
  再定数据与知识层
  再定部署与观测

- 交付校验法:
  能不能演示
  能不能追踪
  能不能评测
  能不能迭代

## 引导项目

- 企业审批助手全栈版
- 研究控制台产品化版
- 多工具工作台 Agent

## 主动练习

- 画一个全栈 Agent 系统图
- 补上:
  前端
  API
  Agent runtime
  工具层
  知识层
  数据层
  观测层
  部署层

## 放行标准

- 能完成一个全栈 Agent 产品原型
- 能说明部署、观测、评测方案
- 能做一次框架选型说明
