# 能力矩阵

## 核心能力维度

| 能力维度 | 初阶 | 中阶 | 高阶 |
| --- | --- | --- | --- |
| Agent 心智模型 | 能解释 Agent 组成 | 能拆解 Agent 系统 | 能设计可迁移的 Agent 架构 |
| LangChain | 能搭基础 Agent | 能接 tools 与 structured output | 能做高效高层抽象与边界控制 |
| LangGraph | 能理解图执行 | 能设计状态与分支 | 能做持久执行、恢复、审批和复杂编排 |
| Deep Agents | 能理解定位 | 能配置复杂任务行为 | 能做 context engineering 与 subagent 拆解 |
| 工具与 MCP | 能接入单个工具 | 能组合多工具与权限 | 能设计可治理的工具生态 |
| RAG 与知识库 | 能做基础检索 | 能做分层知识接入 | 能构建可评测的知识系统 |
| 数据与后端 | 能理解 API / DB 角色 | 能设计存储与服务边界 | 能接真实业务系统 |
| 前端体验 | 能做聊天界面 | 能设计状态与审批体验 | 能设计清晰可控的 Agent UX |
| 评测与观测 | 能看结果与日志 | 能做指标和失败复盘 | 能建立持续评测闭环 |
| 产品化能力 | 能描述场景 | 能定义用户主路径 | 能做完整 Agent 产品方案 |
| 代码理解与审核 | 能看懂单文件职责 | 能解释关键函数与调用链 | 能审查系统逻辑并指出风险与取舍 |

## 课程中的成长标准

- 初阶不是“知道名词”，而是能用自己的话讲清楚
- 中阶不是“做过教程”，而是能迁移到新场景
- 高阶不是“会堆复杂度”，而是能做取舍、治理和审核

## 对本课程学习者的特别要求

针对“有提示词经验、知道 Agent、能描述需求”的学习者，本课程会特别加强下面几条能力：

- 从提示词思维升级到系统设计思维
- 从单轮对话升级到状态化执行思维
- 从“让 AI 输出东西”升级到“让 Agent 在复杂环境里可靠工作”
- 从“让 AI 写代码”升级到“能看懂 AI 写的代码，并做审核判断”
- 从后端思维扩展到前端体验和产品交付思维

## 代码理解能力的具体表现

学习者完成课程后，应逐步具备：

- 看懂某个文件负责什么
- 看懂一个功能由哪些模块协作完成
- 看懂关键函数、关键状态和关键数据流
- 能用自然语言解释代码逻辑
- 能指出代码中哪些部分在实现工具调用、状态管理、审批、检索或前端展示
- 能对 AI 写出的代码做基本审核，而不是只接受结果
