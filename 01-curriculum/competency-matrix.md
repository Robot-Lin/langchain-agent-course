# 能力矩阵

## 核心能力维度

| 能力维度 | 初阶 | 中阶 | 高阶 |
| --- | --- | --- | --- |
| Agent 心智模型 | 能解释 Agent 组成 | 能拆解 Agent 系统 | 能设计可迁移的 Agent 架构 |
| LangChain | 能搭基础 Agent | 能接工具与结构化输出 | 能做高效高层抽象与边界控制 |
| LangGraph | 能理解图执行 | 能设计状态与分支 | 能做持久化、恢复、审批和复杂编排 |
| Deep Agents | 能理解定位 | 能配置复杂任务行为 | 能针对复杂目标做上下文工程与子代理拆解 |
| 工具与 MCP | 能接入单个工具 | 能组合多工具与权限 | 能设计可治理的工具生态 |
| RAG 与知识库 | 能做基础检索 | 能做分层知识接入 | 能构建可评测的知识系统 |
| 数据与后端 | 能理解 API/DB 角色 | 能设计存储与服务边界 | 能搭真实业务系统集成 |
| 前端体验 | 能做聊天界面 | 能设计任务态与审批态 | 能设计可信、透明、可控的 Agent UX |
| 评测与观测 | 能看日志和结果 | 能做指标和失败复盘 | 能建立持续评测闭环 |
| 产品化能力 | 能描述场景 | 能定义用户流程 | 能做完整的 Agent 产品方案 |

## 课程中的成长标准

- 初阶不是“知道名词”，而是能用自己的话讲清楚
- 中阶不是“做过教程”，而是能迁移到新场景
- 高阶不是“会堆复杂度”，而是能做取舍和治理

## 学习者画像对应重点

针对“有提示词经验、知道 Agent、能描述需求”的学习者，课程将重点加强以下内容:

- 从提示词思维升级到系统设计思维
- 从单轮对话升级到状态化执行思维
- 从“让 AI 输出东西”升级到“让 Agent 在复杂环境里可靠工作”
- 从后端思维扩展到前端体验和产品思维
