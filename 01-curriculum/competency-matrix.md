# 能力矩阵

## 核心能力维度

| 能力维度 | 初阶 | 中阶 | 高阶 |
| --- | --- | --- | --- |
| Agent 心智模型 | 能解释 Agent 的基本组成 | 能拆解一个 Agent 系统 | 能设计可迁移的 Agent 架构 |
| LangChain | 能搭最小高层原型 | 能接 tools、structured output、middleware | 能做高层抽象与边界控制 |
| LangGraph | 能理解图执行 | 能设计 state、分支、interrupt | 能做持久执行、恢复、审批与复杂编排 |
| Deep Agents | 能理解产品定位 | 能组织复杂任务与上下文 | 能做 subagent 拆解与 context engineering |
| 工具与 MCP | 能接入单个工具 | 能组合多工具与权限边界 | 能设计可治理的工具系统 |
| RAG 与知识库 | 能做基础检索 | 能做分层知识接入 | 能设计可评测的知识系统 |
| 数据与后端 | 能理解 API / DB 角色 | 能设计存储与服务边界 | 能接真实业务系统 |
| 前端体验 | 能做基础聊天界面 | 能设计状态与审批体验 | 能设计清晰可控的 Agent UX |
| 评测与观测 | 能查看结果与日志 | 能定义指标与失败复盘 | 能建立持续评测闭环 |
| 产品化能力 | 能描述场景 | 能定义主路径和边界 | 能做完整 Agent 产品方案 |
| 代码理解与审核 | 能解释单文件职责 | 能解释主路径和关键调用链 | 能审核系统逻辑并指出风险与取舍 |
| 全项目理解 | 能看懂目录与主要文件 | 能讲清跨文件调用与依赖 | 能解释整个项目如何组织并指出演进方向 |

## 课程中的成长标准

- 初阶不是“知道名词”，而是能用自己的话讲清楚
- 中阶不是“做过教程”，而是能迁移到新场景
- 高阶不是“堆复杂度”，而是能做取舍、治理和审核

## 对本课程学习者的特别要求

针对“有提示词经验、知道 Agent、能描述需求”的学习者，这套课会特别强化：

- 从提示词思维升级到系统设计思维
- 从单轮对话升级到状态化执行思维
- 从“让 AI 输出东西”升级到“让 Agent 在复杂环境里可靠工作”
- 从“让 AI 写代码”升级到“能看懂 AI 写的代码并做审核判断”
- 从“看懂关键代码”升级到“理解整个项目如何被组织和实现”

## 代码理解能力的具体表现

学习者完成课程后，应逐步具备：

- 看懂某个文件负责什么
- 看懂一个功能由哪些模块协作完成
- 看懂关键函数、关键状态和关键数据流
- 能用自然语言解释关键代码逻辑
- 能指出哪些代码在实现工具调用、状态管理、审批、检索或前端展示
- 能对 AI 写出的代码做基本审核，而不是只接受结果

## 全项目理解能力的具体表现

学习者完成课程后，还应逐步具备：

- 看懂整个项目有哪些目录、文件、入口和依赖
- 解释每个主要文件为什么存在、负责什么、不负责什么
- 讲清一条请求如何跨多个文件流动
- 讲清这里具体用了哪个框架能力，为什么这样用
- 指出项目里最脆弱的 2 到 3 个连接点
- 判断哪些地方是真正的主路径，哪些地方是辅助能力
